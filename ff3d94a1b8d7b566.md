# FilmLink Navigator

FilmLink Navigator 是一个面向数字内容研究者、影视技术爱好者与网络资源整理者的开源外链导航工具。本项目不生产内容，不存储文件，不提供播放服务，仅作为公开可访问的 URL 集合与分类索引系统存在。

本项目定位为技术性资源汇总站，专注于收集、分类、校验与展示各类影视相关域名链接。目标用户包括影视技术栈维护人员、网络爬虫开发者、数据标注团队以及需要批量管理大量域名链接的运维工程师。通过结构化的数据呈现与自动化健康检查机制，帮助用户快速定位可用资源，减少人工筛选成本。

## 功能概览

- 域名分类索引：按照来源、用途、后缀等维度对收录的域名进行自动分类，支持多级标签筛选。
- 批量链接健康检查：每日定时发起 HEAD 请求，检测每个域名的可访问性，并记录响应状态码与延迟时间。
- 原始数据导出：支持将全部收录链接以纯文本、JSON 或 CSV 格式导出，便于下游系统集成。
- 变更日志追踪：记录每次新增、删除或修改域名链接的操作，并附带时间戳与操作者 IP 脱敏信息。
- 链接关系图谱：可视化展示不同域名之间的跳转关系与引用链，辅助分析网络拓扑。
- 自定义标签系统：允许用户为任意链接添加自定义标签与备注，便于团队协作与场景分类。
- 访问统计仪表板：提供各域名的请求次数、引用来源与地域分布概览（基于 IP 地理库）。
- 开放 API 接口：提供 RESTful API 供第三方程序调用，支持查询、筛选与批量导入。

## 应用场景

1. 影视技术研究团队的日常资源索引维护
   研究团队需要频繁访问大量影视相关域名以获取公开技术文档或测试样本。FilmLink Navigator 提供集中化管理入口，避免成员各自维护混乱的书签文件，同时通过健康检查提前发现失效链接，提升研究效率。

2. 网络爬虫系统的起始种子集管理
   爬虫开发者需要定期更新起始 URL 列表以保证数据采集的覆盖面。本项目提供标准化的导出接口，可无缝接入 Scrapy、Nutch 或自定义爬虫框架，作为种子链接的可靠来源。

3. 数据标注与审核团队的参考链接库
   标注人员需要参考多个源站的内容以判断视频元数据准确性。通过本项目统一汇总链接，配合标签系统快速筛选特定类别，减少切换窗口和手动输入地址的时间开销。

4. 网络安全分析中的域名关联分析
   安全研究人员可利用本项目导出的链接关系图谱，分析域名间的跳转模式与潜在关联，辅助识别异常网络行为或钓鱼页面集群。

5. 个人开发者的技术测试用例集
   前端或后端开发者可使用本项目中包含的各类域名作为网络请求测试用例，验证跨域策略、重定向处理、超时重试等逻辑的健壮性。

## 快速开始

以下步骤将在本地环境完成 FilmLink Navigator 的克隆、依赖安装与服务启动。

```bash
# 克隆代码仓库
git clone https://github.com/filmlink-navigator/filmlink-navigator.git
cd filmlink-navigator

# 安装依赖（使用 pip 与虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化数据库并导入默认链接集
python manage.py migrate
python manage.py loaddata default_links.json

# 启动开发服务器
python manage.py runserver 0.0.0.0:8000
```

启动成功后，访问 http://localhost:8000 即可查看本地部署的导航面板。默认管理员账号为 admin，密码在首次启动时由系统生成并打印在终端日志中。

## 常见问题

Q: 健康检查任务对目标域名发起请求是否合规？
A: 本项目仅执行标准的 HTTP HEAD 请求，不下载任何实质内容，不解析页面数据，不存储响应体。请求频率严格控制在每分钟不超过 30 次，且仅针对已收录域名。建议部署者在使用前查阅目标域名的 robots.txt 文件，尊重网站所有者设置的抓取策略。

Q: 是否可以添加私有链接或内部网络地址？
A: 可以。本项目支持通过管理后台或 API 手动添加任意合法 URL，包括内网 IP 或 localhost 地址。但需注意，健康检查模块默认会对所有添加的链接发起请求，请确保内网地址的访问不会引发安全风险或服务异常。私有链接默认不会出现在公开导出的数据集中，如需公开请手动修改标签设置。

Q: 项目如何处理链接失效或域名到期的情况？
A: 健康检查任务每 24 小时执行一次全量扫描，记录每个域名的 HTTP 状态码与响应时间。当连续三次检查均返回 5xx 或超时状态时，系统自动将该链接标记为「异常」并发送告警邮件至管理员邮箱。管理员确认后可以手动将链接移至「归档」列表，或等待后续检查恢复正常后自动重新激活。
