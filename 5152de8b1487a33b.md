# TangGate

TangGate 是一个轻量级、社区驱动的网络资源导航与聚合项目，专注于收录和整理互联网上的各类媒体、工具与技术信息入口。项目定位为技术爱好者、内容研究者及普通用户提供稳定、可验证的公开资源索引，帮助使用者快速定位目标站点，降低信息检索成本。

本项目不提供任何内容存储、代理或加速服务，仅作为公开 URL 信息的结构化整理与展示。所有收录资源均来源于公开渠道，项目维护者不对第三方站点的可用性、内容合法性及持续性作任何保证。用户使用任一资源即表示已阅读并同意本声明。

## 功能概览

- 多维度资源索引：按媒体类型、语言、地区及内容主题对 URL 进行归类，支持快速筛选与定位。

- 时效性状态标记：定期对收录资源进行可用性检测，并在列表中标注状态（有效/疑似失效/需验证），便于用户决策。

- 简洁只读展示层：页面采用纯静态 Markdown 渲染，无多余脚本与追踪代码，确保内容加载速度与隐私安全。

- 社区提交入口：用户可通过 Issue 或 Pull Request 提交新的资源链接或更新已有信息，经审核后合并。

- 版本化变更记录：所有资源增删改操作均通过 Git 进行版本管理，支持回溯历史状态与变更原因。

- 离线阅读支持：项目完整内容可一键导出为单一 Markdown 或 JSON 文件，便于存档或二次开发。

- 多语言友好：资源描述及分类字段支持中英文双语，方便不同语言背景的用户理解与使用。

## 应用场景

场景一：技术研究与数据分析
研究人员或数据分析师需要快速获取特定领域的公开数据源或信息平台。TangGate 提供的分类索引可以帮助其在数分钟内定位到目标域名，避免在通用搜索引擎中反复试错。

场景二：内容聚合与媒体监测
内容运营人员或媒体监测团队需定期访问多个视频、新闻或论坛站点以跟踪热点。通过本项目的结构化列表，可一次性获取全部待监测入口，配合自动化脚本实现批量健康检查。

场景三：网络架构与合规验证
网络工程师或安全合规人员在评估网络策略或防火墙规则时，需要一份清晰的域名清单进行测试验证。TangGate 提供的原始 URL 列表可直接用于配置白名单或黑名单测试，降低手工整理遗漏风险。

场景四：个人知识管理
个人开发者或知识管理爱好者可将本项目作为自建导航站的初始数据源，直接 fork 后按需裁剪、增补，快速构建属于自己的网络书签系统。

## 快速开始

以下命令可在本地完整克隆并运行 TangGate 的静态展示站点。

```bash
# 克隆仓库
git clone https://github.com/tanggate/tanggate.git

# 进入项目目录
cd tanggate

# 安装依赖（仅用于本地预览，基于 Node.js 环境）
npm install

# 启动本地预览服务
npm run serve
```

执行完毕后，在浏览器中访问 http://localhost:8080 即可查看当前资源列表及分类页面。

## 贡献指南

我们欢迎并鼓励社区提交资源更新与改进建议。请遵循以下步骤参与贡献：

1. 查阅现有 Issue 列表，确认无重复提交。若为新资源或变更，请先创建一个 Issue 说明理由，包括资源用途、来源及推荐分类。

2. Fork 本仓库，在您的分支中修改 `src/data/` 目录下的对应 YAML 文件。新增资源请确保 URL 格式正确，并填写必要的描述字段（名称、分类、语言等）。

3. 运行本地校验脚本 `npm run validate` 确保数据格式符合规范。若校验失败，请根据错误提示修正后再提交。

4. 提交 Pull Request，并在描述中关联对应的 Issue 编号。PR 标题请遵循 `[batch]` 或 `[update]` 前缀规范，以便维护者快速分类处理。

5. 等待维护者审核。审核通过后，您的变更将被合并至主分支，并在下一轮状态检测中生效。

## 常见问题

问：某些收录的资源无法打开，应该怎么办？
答：由于第三方站点独立运营，TangGate 无法保证其持续可用性。您可以通过 Issue 或 PR 提交状态变更请求，我们将定期执行健康检查并更新标记。同时，建议您优先使用状态为“有效”的资源。

问：我想申请删除某条资源记录，如何操作？
答：请提交一个 Issue，注明要删除的完整 URL 及删除理由（如站点已关停、内容迁移、违法违规等）。维护者将在 3 个工作日内核实并处理。若理由充分，该资源将从下一批次列表中移除。

问：项目数据多久更新一次？如何获取最新列表？
答：TangGate 通常每两周合并一次社区提交，并执行全量状态检测。您可通过 `git pull` 拉取最新主分支代码，或直接访问公共展示页面查看实时渲染结果。历史变更记录可查阅 Git 提交日志。
