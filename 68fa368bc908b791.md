# FilmNav

FilmNav 是一个面向数字内容研究者、影视技术爱好者和自媒体的轻量级导航与资源聚合工具。本项目并非提供具体影视内容，而是对互联网上散落的优质影视相关站点进行结构化梳理与分类索引，帮助用户在繁杂的网络环境中快速定位到可用的影视资源入口、信息查询服务和行业动态来源。

项目定位为“影视外链的中立索引库”，不存储任何版权文件，不提供播放或下载服务，仅记录公开可访问的 URL 信息。目标用户包括影视行业从业者、内容运营人员、影评人以及需要大量影像素材进行研究的设计师与视频创作者。

## 功能概览

- 动态站点存活监测：系统每日对收录的 URL 进行 HTTP 状态检查，自动标记不可用站点，确保索引库的时效性。
- 多维度分类筛选：按地区、语言、内容类型（电影、剧集、纪录片、动画）和访问协议（HTTP/HTTPS）对资源进行分类。
- 快速关键词检索：支持对站点标题、描述和标签进行全文搜索，响应时间低于 200 毫秒。
- 用户自定义收藏夹：注册用户可将常用站点加入个人收藏，并自定义分组标签。
- 站点信息详细展示：每个资源条目包含域名、备案信息（如有）、内容简介、更新频率和语言选项。
- 反钓鱼与安全提示：对已知风险域名进行标注，并通过算法提示用户注意非标准端口或异常跳转行为。
- 开放 API 接口：提供 RESTful API 供第三方开发者批量获取站点列表，支持 JSON 和 XML 格式输出。
- 访问统计与热度排行：基于点击量和用户评价生成周榜与月榜，辅助发现高质量资源。

## 应用场景

场景一：影视自媒体运营者寻找素材源
运营者需要快速获取不同国家或地区的影视预告片、海报和剧照用于内容制作。通过 FilmNav 的“地区筛选”和“媒体类型”组合查询，可以在数秒内获得一批高访问量的站点列表，显著提升素材搜集效率。

场景二：学生研究者进行影视传播学课题调研
研究者需要分析国内小型影视站点的生存状况和访问特征。利用 FilmNav 的存活监测数据和 API 接口，研究者可以批量导出站点元数据，进行数据可视化分析，无需手动逐个测试域名可用性。

场景三：普通影迷寻找冷门影片信息
当主流平台无法检索到某部冷门影片的演职员表或发行信息时，用户可通过 FilmNav 的“特殊资源”分类，找到专注于老电影、独立电影或地区特色影片的信息站，从而补全检索结果。

场景四：前端开发者学习外部链接管理方案
开发者可参考 FilmNav 的站点监测、缓存更新和异常处理机制，将其作为微服务架构中“外部依赖治理”的教学案例，或基于开放 API 构建自己的导航前端。

## 快速开始

以下步骤适用于 Linux / macOS / Windows WSL 环境，需提前安装 Git、Node.js（v18 以上）和 pnpm。

```bash
# 克隆代码仓库
git clone https://github.com/filmmap/filmnav.git
cd filmnav

# 安装项目依赖
pnpm install

# 配置环境变量（复制示例配置）
cp .env.example .env.local

# 初始化数据库（SQLite）
pnpm run db:init

# 启动开发服务器（默认监听 3000 端口）
pnpm run dev
```

访问本地服务 `http://localhost:3000` 即可查看前端界面。若需构建生产版本并启动，请执行：

```bash
pnpm run build
pnpm run start
```

## 贡献指南

我们欢迎社区开发者提交改进建议、新增站点索引或完善文档。请遵循以下步骤参与贡献：

1. 查阅问题列表：访问 GitHub Issues 板块，优先认领标记为 `help wanted` 或 `good first issue` 的任务，避免重复工作。

2. 新增站点索引：若想推荐新的影视站点，请先在 `docs/site-request-template.md` 中填写站点名称、URL、分类标签和简短描述，并提交 Pull Request。项目维护者将在 48 小时内审核存活状态和内容合规性。

3. 代码提交规范：使用 Conventional Commits 格式（如 `feat: 添加日本动画分类筛选` 或 `fix: 修复监测超时导致进程崩溃的问题`）。提交前请运行 `pnpm run lint` 和 `pnpm run test` 确保代码质量。

4. 文档更新：任何功能性变动必须同步修改对应文档章节，特别是 API 响应格式变更和配置项增删。文档使用 Markdown 编写，存放在 `/docs` 目录下。

5. 行为准则：请遵守贡献者公约，保持友善且专业的沟通。所有讨论和代码审查均在 GitHub 上进行公开记录。

## 常见问题

问：本站点是否提供电影下载或在线播放服务？

答：不提供。FilmNav 严格定义为外链索引工具，所有链接指向的是第三方公开站点。我们不对第三方站点的内容合法性、安全性或可用性承担任何责任。用户应遵守当地法律法规并自行判断访问风险。

问：我发现收录的某个站点已经无法访问或内容异常，应如何报告？

答：您可以通过以下方式反馈：在站点详情页点击“报告失效”按钮（需登录），或在 GitHub Issues 中提交失效报告，附带站点 URL 和截图。系统每日也会自动扫描，但人工反馈能帮助我们更快更新状态。

问：能否申请将自己的站点从索引中移除？

答：可以。站点所有者或版权方代表请发送邮件至 `filmnav-admin@example.com`（请替换为实际管理员邮箱），邮件中需包含域名所有权证明（如 DNS TXT 记录值或网站根目录上传验证文件）。我们会在 3 个工作日内处理移除请求。
