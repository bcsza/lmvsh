# HyperLink Hub

HyperLink Hub 是一个面向数字内容研究者、网络资源整理者及自动化采集系统维护人员的轻量级外链资源聚合平台。项目定位为技术导向的公开 URL 资产索引仓库，旨在解决分散、易失效、缺乏版本追踪的链接管理问题，提供一套结构化、可校验、可离线使用的链接数据集与配套工具。目标用户包括数据标注团队、SEO 技术运维、学术资源爬取开发者以及任何需要系统化维护外部链接清单的工程人员。

本项目不提供网页爬取或自动化访问功能，仅作为 URL 元数据的整理与分发中心，所有收录链接均附带基础上下文标签、入库时间戳及可用性探测状态标记。通过标准化的 JSON Schema 输出与 Git 版本控制，实现对链接资源的全生命周期追踪，有效降低链接腐化对业务系统的影响。

## 功能概览

- **结构化链接台账**：所有收录 URL 按照来源类别、域名后缀、业务属性进行多级标签分类，支持快速筛选与导出。
- **可用性自动校验**：每日定时执行 HEAD 请求探测，标记链接状态为有效、可疑或失效，并记录响应时间与 HTTP 状态码。
- **版本化变更日志**：每一次链接增删改操作均生成 CHANGELOG 条目，支持回溯任意历史版本的完整链接集合。
- **多格式数据导出**：内置转换脚本支持输出为 CSV、JSON Lines 或纯文本列表，便于下游系统直接消费。
- **自定义标签系统**：用户可对任意链接附加自定义键值对元数据，如所属项目代号、预期用途、更新周期等。
- **链接关系图谱**：提供简单的域名聚合与共现分析，可视化展示不同资源之间的引用关联强度。
- **健康度看板**：通过命令行工具生成简洁的统计报告，包含总链接数、失效比例、响应时间分位数等关键指标。
- **Webhook 通知**：当检测到批量链接状态异常变化时，支持通过自定义回调地址推送告警事件。

## 应用场景

- **数据采集管道前置校验**：在启动大规模分布式爬虫任务前，通过 HyperLink Hub 的可用性探测结果筛除已失效的目标域名，避免无效请求浪费网络资源与计算资源，提升采集任务的整体成功率。
- **学术文献外链附录整理**：研究团队在撰写系统综述或技术报告时，可将文中引用的所有网络资源统一录入 Hub 进行集中管理，借助版本日志记录每次访问时间与状态，确保评审阶段引用来源的可追溯性与可复现性。
- **企业内部知识库外链治理**：企业维基或文档中心大量嵌入外部参考链接，随着时间推移大量链接失效影响阅读体验。运维团队可定期同步 HyperLink Hub 的检测结果，批量更新或清理知识库中的坏死链接，维持文档质量。
- **开源项目依赖资源镜像筛选**：开源软件在构建过程中可能需要从外部源下载依赖包或预训练模型，使用 Hub 维护的备选链接列表可实现自动故障切换，当主链接不可用时快速切换至镜像源，提高构建稳定性。

## 快速开始

以下命令演示如何在本地环境克隆仓库、安装依赖并启动基础服务。

```bash
# 克隆仓库至本地
git clone https://github.com/hyperlink-hub/public-index.git
cd public-index

# 安装 Python 依赖（推荐使用虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 执行链接校验并生成报告
python cli.py validate --input data/urls.json --output reports/status.json
```

如需启动 Web 可视化看板（开发模式），请执行：

```bash
python cli.py serve --port 8080
```

访问 http://localhost:8080 即可查看当前链接总览与状态分布图表。

## 常见问题

**Q: 为什么检测结果中频繁出现超时或连接拒绝，但浏览器可以正常访问？**

A: 此类情况通常由目标站点的反爬策略或防火墙规则导致。HyperLink Hub 的探测模块默认使用简单的 User-Agent 头且不携带 Cookie，部分站点会直接拒绝服务。您可以调整配置中的 `headers` 参数模拟真实浏览器指纹，或增加 `delay` 参数降低请求频率。另外，请确认您的运行环境网络出口 IP 未被目标站点列入黑名单。

**Q: 如何批量导入大量历史链接，而不是逐条手动添加？**

A: 项目支持批量导入功能。请按照 `data/urls.json` 中的 Schema 结构准备一个 JSON 数组文件，每个元素需包含 `url` 和 `tags` 字段。然后在命令行执行 `python cli.py import --source your_file.json --merge`，系统会自动合并去重并触发初始可用性探测。导入前建议先使用 `--dry-run` 参数进行模拟执行以校验数据格式。

**Q: 自定义标签的命名是否有特殊限制？**

A: 标签名称仅支持小写字母、数字、下划线和中划线，长度不超过 32 个字符。标签值（即键值对中的 value）无严格格式限制，但建议保持为字符串类型以便于一致处理。所有自定义标签均会记录在 `data/tags.yaml` 的 `custom` 节点下，请避免与系统预置标签（如 `status:valid`、`category:media` 等）发生键名冲突。
