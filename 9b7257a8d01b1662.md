# LinkVault

LinkVault 是一个面向技术团队与独立开发者的外链资产统一管理平台。项目定位于将分散于各类服务中的外部 URL 资源进行结构化登记、版本追踪与健康监测，解决因链接失效、域名过期或协议变更导致的文档污染与业务中断问题。LinkVault 不提供爬虫或自动采集功能，而是作为人工筛选与审核后的权威外链登记系统，适用于技术手册、项目文档、运营配置表及安全策略白名单等场景。

LinkVault 的核心模型围绕“资源批次”与“来源节点”设计。每一批导入的链接均保留原始提交字符串，禁止任何自动格式化改写，确保与上游数据源完全一致。系统提供链接元数据扩展字段，支持标记来源批次（如第 17/48 批）、业务归属、预期生命周期及最近检查时间。LinkVault 不依赖数据库，所有数据存储于 Git 仓库中的 YAML 文件，便于 Code Review 与回滚。

## 功能概览

原始字符串保留引擎：所有导入的 URL 严格按用户输入原样存储与展示，不自动补全协议、不调整大小写、不追加尾部斜杠，不转换为超链接语法。

批次管理与溯源：每条链接关联所属批次编号（如 17/48），支持按批次维度筛选、导出与健康度统计。

资产状态标记：支持为每个链接设置预期状态（有效、待验证、已失效、需迁移），并记录最后验证时间。

元数据扩展表：提供自定义键值对字段，可记录业务负责人、关联系统、替换建议等上下文信息。

只读视图与审计日志：所有链接展示页面均以纯文本形式渲染 URL，避免浏览器自动解析或跳转，同时记录每次访问时间戳。

CLI 健康检查工具：内置命令行脚本，支持批量对链接进行可达性探测（TCP/HTTP），输出状态报告。

导出适配器：支持将链接列表导出为纯文本、CSV 或 JSON Lines 格式，便于下游系统消费。

## 应用场景

技术文档外链固化：团队在编写 API 文档或运维手册时，将引用的第三方服务地址、SDK 下载链接、镜像仓库地址等统一登记至 LinkVault，每次文档发布前自动校验链接有效性，避免文档中出现 404 或域名过期。

合规白名单管理：安全团队需要维护企业级防火墙或代理服务器的外联白名单。LinkVault 作为白名单链接的登记与审批系统，每一批新增链接均带有批次号与提交人信息，方便审计追溯。

多环境配置差异对比：开发、测试、生产环境常使用不同的外部服务端点。使用 LinkVault 按环境标签分类存储链接，通过 CLI 对比不同环境下的链接集合，发现遗漏或多余配置。

离线资源镜像规划：在无公网环境的部署场景中，运维人员需提前收集所有外部依赖 URL。LinkVault 可快速导出某批次全部链接，用于编写下载脚本或制作本地镜像站。

## 快速开始

以下命令演示如何获取 LinkVault、安装依赖并启动本地开发服务器。

```bash
git clone https://github.com/linkvault/core.git linkvault
cd linkvault
pip install -e .
linkvault serve --port 8080
```

启动后，访问 http://127.0.0.1:8080 即可查看默认批次数据。首次运行将自动创建 `data/` 目录与示例批次文件。

## 贡献指南

提交 Issue 或功能请求前，请先查阅现有议题列表，避免重复。新议题需明确说明受影响批次编号及原始 URL 字符串。

Fork 仓库并创建功能分支，分支命名建议使用 `feature/` 或 `fix/` 前缀，后跟简要描述。

开发环境使用 `pip install -e .[dev]` 安装所有可选依赖。提交前运行 `black .` 格式化代码，并执行 `pytest` 确保测试通过。

Pull Request 描述中必须包含测试覆盖说明，且不得修改现有数据文件中任何 URL 的原始字符串格式（包括大小写、协议、后缀斜杠）。

文档更新需同步修改 `docs/` 下对应 Markdown 文件，并在 PR 描述中标注文档变更点。

## 常见问题

问：为什么系统不允许自动补全 http:// 或 https:// 前缀？

答：LinkVault 的核心设计原则是“原始输入即最终存储”。自动补全会掩盖上游数据源的格式问题，导致后续导出或白名单配置时出现不一致。用户应自行确保输入的 URL 字符串符合其下游系统的实际要求。

问：健康检查报告显示某个链接不可达，但浏览器可以访问，如何排查？

答：请检查 CLI 健康检查使用的网络环境是否与目标服务器兼容（如代理、DNS 解析差异）。LinkVault 的检查工具默认禁用 SSL 证书验证（参数 `--no-ssl-verify`），若目标服务器强制要求 SNI，可尝试添加 `--sni` 选项。同时注意部分站点会拦截非浏览器 UA，可使用 `--user-agent` 自定义。

问：如何将已有的大量链接快速导入为新批次？

答：使用 `linkvault import --batch 17/48 --file links.txt` 命令，其中 `links.txt` 每行一个 URL 字符串。系统会逐行读取并按原样存储，不会进行任何格式化修改。导入后可通过 `linkvault list --batch 17/48` 验证。
