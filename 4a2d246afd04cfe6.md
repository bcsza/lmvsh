# LinkPilot

LinkPilot 是一个面向开发者和技术内容创作者的开源外链资源聚合平台，旨在通过结构化数据与自动化校验机制，解决技术文档中外部链接失效、域名迁移、协议不统一等管理难题。项目定位为“技术资源的元数据库”，不生产内容，但提供对分散于网络各处的优质技术站点的索引、分类、状态监控与访问引导。

目标用户包括开源项目维护者、技术博客作者、在线教育平台运营者以及企业内部知识库管理员。LinkPilot 通过预设的规则引擎与人工审核相结合的混合模式，确保收录的每一个资源链接均具备可用性与领域相关性，降低技术信息检索中的噪声干扰，提升研发团队的协作效率。

## 功能概览

- **多协议链接归一化存储**：支持 HTTP、HTTPS、裸域名、带路径与查询参数的 URL 原样入库，保留用户原始输入格式，不做任何隐性改写，确保审计追溯能力。

- **自动化可用性探测**：每日定时任务对已收录链接进行 HEAD/GET 请求探测，标记响应状态码、响应时间与重定向链，生成可用性报告。

- **分类标签与全文检索**：基于正则表达式与关键词库自动为链接打上技术栈标签，并支持对 URL、标题、描述、标签的快速全文检索，便于定位特定资源。

- **访问统计与热力排行**：记录每个外链的点击次数、最近访问时间与来源 Referrer，生成周/月维度的访问热度图表，辅助识别高价值资源。

- **结构化导出接口**：提供 RESTful API 与 CSV/JSON 批量导出能力，支持将选中的资源列表一键嵌入到 MkDocs、VuePress、Hugo 等静态站点生成器中。

- **变更通知订阅**：当某个链接出现状态码变化、证书过期或域名 whois 信息变更时，通过 Webhook 或邮件通知订阅者，实现异常及时感知。

## 应用场景

- **开源项目 README 外链维护**：开源项目维护者可使用 LinkPilot 管理项目文档中引用的所有第三方依赖文档、教程、API 参考链接，定期自动检查链接有效性，避免因外链失效导致用户困惑。

- **技术博客资源聚合页构建**：技术博主通过 LinkPilot 收集、分类并展示自己推荐的工具库、在线课程、社区论坛，配合访问统计功能了解读者偏好，优化推荐策略。

- **企业知识库外部引用治理**：企业内部 Wiki 或 Confluence 空间中存在大量外部技术链接，使用 LinkPilot 进行统一登记与健康检查，确保员工培训材料和操作手册中的引用始终可达。

- **在线教育平台课程资料索引**：编程教学平台利用 LinkPilot 为每门课程建立独立的资源清单，包括代码沙盒、官方文档、习题解答参考，并支持按课程编号快速筛选与导出。

## 快速开始

以下步骤帮助您在本地环境快速启动 LinkPilot 服务。

```bash
# 克隆项目仓库
git clone https://github.com/linkpilot/linkpilot.git
cd linkpilot

# 安装 Python 依赖（推荐使用虚拟环境）
python -m venv venv
source venv/bin/activate  # Windows 下使用 venv\Scripts\activate
pip install -r requirements.txt

# 初始化 SQLite 数据库结构
python manage.py migrate

# 导入示例资源数据（包含本批次 URL）
python manage.py loaddata fixtures/initial_links.json

# 启动开发服务器
python manage.py runserver --host 0.0.0.0 --port 8000
```

访问 http://localhost:8000 即可进入 Web 管理界面。默认管理员账户为 `admin:admin123`，首次登录后请立即修改密码。

## 项目结构

```
linkpilot/
├── backend/                        # 后端核心代码（Python + Django）
│   ├── api/                        # RESTful API 路由与序列化器
│   ├── checker/                    # 链接可用性探测模块（含异步 Worker）
│   │   ├── probe.py                # 实现 HTTP/HTTPS 探测及重定向跟踪
│   │   └── scheduler.py            # 基于 APScheduler 的定时任务配置
│   ├── core/                       # 项目配置与全局常量
│   │   ├── settings.py             # Django 设置（含数据库、缓存、日志）
│   │   └── urls.py                 # 根路由分发
│   ├── models/                     # ORM 模型定义（Link, Tag, ClickLog, ProbeResult）
│   ├── services/                   # 业务逻辑层（导入导出、统计计算）
│   └── utils/                      # 工具函数（URL 规范化、域名解析辅助）
├── frontend/                       # 管理界面（Vue 3 + Vite）
│   ├── src/
│   │   ├── components/             # 可复用 UI 组件（表格、图表、筛选器）
│   │   ├── views/                  # 页面级视图（仪表盘、链接列表、详情）
│   │   └── stores/                 # Pinia 状态管理（当前筛选、分页、主题）
├── scripts/                        # 运维与辅助脚本
│   ├── init_db.py                  # 首次初始化数据库并创建超级用户
│   └── seed_links.py               # 从 CSV 或 JSON 批量导入链接
├── tests/                          # 单元测试与集成测试
│   ├── test_checker.py             # 探测逻辑的边界条件测试（超时、重定向、SSL）
│   └── test_api.py                 # API 端点状态码与数据完整性验证
├── docs/                           # 完整文档（Markdown + MkDocs 配置）
├── requirements.txt                # Python 生产依赖列表
├── requirements-dev.txt            # 开发额外依赖（pytest, black, flake8）
├── docker-compose.yml              # 本地开发与生产容器编排（含 Redis）
├── Dockerfile                      # 基于 slim-buster 镜像的多阶段构建
├── .env.example                    # 环境变量模板（数据库连接、密钥、Webhook 地址）
├── .gitignore                      # 忽略虚拟环境、日志文件、本地配置
└── README.md                       # 本文件
```

## 常见问题

**Q: 为什么我添加的裸域名（如 example.com）在列表中显示正常，但探测状态总是标记为失败？**

A: 默认探测使用 HTTP 协议，但部分裸域名仅支持 HTTPS 且未配置自动重定向。您可以在添加链接时显式指定协议前缀（如 https://example.com），或在管理后台的“探测设置”中开启“自动尝试 HTTPS 回退”选项。此外，请确认您的服务器网络环境允许向外发起 TCP 连接，某些企业防火墙可能限制出口流量。

**Q: 数据量达到多少时建议从 SQLite 迁移到 PostgreSQL？**

A: 当链接记录超过 10 万条，且每日探测任务超过 5000 次时，SQLite 的写入锁竞争可能导致探测任务积压。此时推荐使用 PostgreSQL 作为生产数据库，并配合 Redis 缓存热点统计结果。项目已提供 `settings.py` 中的数据库连接字符串切换支持，只需修改环境变量 `DATABASE_URL` 即可无缝迁移，无需改动业务代码。

**Q: 如何让 LinkPilot 在启动时自动加载本批次或其他批次的资源列表？**

A: 您可以将资源列表整理为 CSV 文件（包含 `url`, `title`, `category`, `tags` 列），然后使用 `python scripts/seed_links.py --file your_list.csv --batch 21` 命令进行批量导入。该脚本会自动校验 URL 格式并跳过重复项，同时为每条记录打上批次标签，便于后续筛选与管理。
