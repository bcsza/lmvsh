# QingNiao Discovery

QingNiao Discovery is a curated technical resource aggregation and navigation system designed for developers, researchers, and IT professionals who need efficient access to high-quality external tools, documentation, and service endpoints. Unlike general-purpose bookmark managers or search engines, this project focuses on maintaining a stable, well-categorized, and continuously updated index of specialized web resources that are frequently used in cloud infrastructure, network diagnostics, and cross-platform development workflows.

The project addresses the common pain point of scattered bookmarks, outdated links, and the lack of contextual metadata around external resources. It provides a structured catalog with availability monitoring, usage notes, and tag-based filtering, enabling teams to maintain a shared knowledge base of valuable external references without relying on proprietary cloud services. Whether you are setting up a new development environment, troubleshooting production issues, or researching emerging technologies, QingNiao Discovery reduces the time spent on link discovery and validation.

## 功能概览

- **Categorized Resource Indexing** – Organize external URLs into logical groups such as API gateways, documentation mirrors, package registries, and diagnostic tools, with support for custom tags and hierarchical folders.

- **Automated Availability Health Checks** – Periodically probe each registered endpoint with configurable timeouts and retry policies, marking status as online, degraded, or unreachable with historical uptime trends.

- **Metadata Enrichment Pipeline** – Automatically fetch and cache HTTP response headers, TLS certificate validity periods, and server fingerprint information for each resource, aiding security audits and compliance checks.

- **Full-Text Search and Filtering** – Provide instant search across resource names, descriptions, tags, and domain components, with faceted filters for status, category, and last-updated time.

- **Team Collaboration Workspace** – Support shared collections with role-based access control, change logging, and comment threads for each resource, enabling team members to annotate usage experiences and internal notes.

- **RESTful API and Webhook Integration** – Expose all indexed data via a versioned JSON API, with outgoing webhook triggers for status change events, allowing integration with monitoring dashboards and alerting systems.

- **Data Export and Backup** – Offer periodic full exports in JSON, YAML, and Markdown table formats, with optional scheduled backups to local storage or remote object stores.

- **Resource Usage Analytics** – Collect anonymous usage metrics such as click counts, search terms, and access latency, presented in an aggregated dashboard to identify the most valuable resources and potential dead links.

## 应用场景

**Development Environment Bootstrapping** – When a new team member joins a project, they can use QingNiao Discovery to quickly locate all required external service endpoints, internal documentation mirrors, and testing sandboxes, eliminating the need to ask colleagues or search through outdated wiki pages.

**Production Incident Diagnosis** – During an outage or performance degradation, engineers can navigate to the network diagnostics category to find alternative DNS resolvers, latency measurement endpoints, and status pages of upstream dependencies, all pre-validated and annotated with historical reliability data.

**Compliance and Security Auditing** – Security officers can regularly review the resource list to verify that all external URLs use appropriate TLS protocols, valid certificates, and are hosted on approved domains, with automated alerts for any expiring certificates or unexpected header changes.

**Documentation and Training Material Maintenance** – Technical writers and trainers can embed QingNiao Discovery links directly into runbooks, course slides, and internal knowledge bases, ensuring that all referenced external resources are centrally maintained and updated without modifying multiple documents.

**Offline and Air-Gapped Environment Mirroring** – For organizations with restricted internet access, administrators can use the export feature to generate a static snapshot of all resource metadata and then synchronize approved endpoints to internal mirrors, preserving the structured index even without live connectivity.

## 快速开始

Clone the repository, install dependencies, and start the development server with the following commands:

```bash
git clone https://github.com/qingniao-dev/discovery.git
cd discovery
npm install --production=false
npm run build
npm run start:dev
```

For production deployment, set the following environment variables before running the start script:

```bash
export NODE_ENV=production
export DATABASE_URL=postgresql://user:pass@localhost:5432/discovery
export REDIS_URL=redis://localhost:6379
export CHECK_INTERVAL_MS=3600000
npm run start:prod
```

The service will be available at http://localhost:3000 by default, with the API root at http://localhost:3000/api/v1. Use the built-in admin account creation command to set up the first administrator user:

```bash
npm run create-admin -- --email admin@example.com --password securePassword123
```

## 常见问题

**Q: How often are health checks performed, and can I customize the interval per resource?**

A: By default, the system performs a health check on every registered resource every 60 minutes. You can override this globally via the `CHECK_INTERVAL_MS` environment variable. For per-resource customization, use the `checkInterval` field in the resource creation or update payload, which accepts a value in milliseconds between 300000 (5 minutes) and 86400000 (24 hours). Note that intervals shorter than 5 minutes are subject to rate limiting to avoid overloading target servers.

**Q: Does QingNiao Discovery store any sensitive information from the probed endpoints?**

A: The health check worker only records HTTP status codes, response time, and a truncated preview of the response body (first 1024 bytes) for debugging purposes. It does not store cookies, authentication headers, or full response payloads unless explicitly enabled via the `DEBUG_RESPONSE_CAPTURE` flag, which is strongly discouraged in production. All captured metadata is encrypted at rest using AES-256-GCM with a key derived from the `ENCRYPTION_SECRET` environment variable. You can disable response body capture entirely by setting `CAPTURE_RESPONSE_BODY=false`.

**Q: How can I migrate data from an existing bookmark manager or spreadsheet?**

A: The system provides an import endpoint that accepts CSV, JSON, and Netscape HTML bookmark formats. Use the admin dashboard or the `POST /api/v1/import` endpoint with the appropriate `Content-Type` header. For custom formats, you can write a transformation script that outputs the standard JSON schema documented in the API reference. The import process validates all URLs, deduplicates entries based on normalized domain and path, and preserves any folder structure as tags.
