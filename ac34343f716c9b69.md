# VideoLink Navigator

VideoLink Navigator 是一个面向视频内容创作者、流媒体技术研究者以及影视资源整合开发者的高质量视频链路与媒体资源导航项目。项目定位为技术化外链聚合与分类索引系统，目标用户包括视频站运维人员、内容聚合工具开发者、媒体数据分析师以及需要系统化访问流媒体工具链的技术人群。

本项目不存储、不托管、不转发任何视频文件或媒体内容，仅提供外部公开资源的结构化导航索引。通过对视频域名、工具站、测试源与媒体样本库的整理归类，显著降低开发者在视频链路调试、播放源测试、清晰度对比与流媒体协议分析等环节中的信息检索成本，提升技术验证效率。

## 功能概览

- 视频链路分类索引：按媒体类型、域名归属与协议特征对收录资源进行一级分类，便于按场景快速定位。

- 域名状态快速检测：提供脚本化检测思路，用户可对列表中各域名进行连通性、响应时间与可用性验证。

- 媒体格式测试样本引用：汇集多种封装格式与编码参数的公开测试链接，用于播放器兼容性测试。

- 流媒体协议对照导航：支持对 HLS、DASH、RTMP 等常见协议的测试端点进行归类与注释。

- 外链变更追踪提醒：通过版本化更新维护链接有效性，每次发布均标注变动记录。

- 第三方工具链推荐：收录视频转码、截图、元数据分析等辅助工具官方入口。

- 多级标签筛选机制：每条链接附带功能标签，便于按分辨率、编码、来源、地区等维度过滤。

- 贡献者共享书签池：支持社区提交新链接，经审核后纳入索引库，形成持续生长的导航生态。

## 应用场景

场景一：视频播放器开发调试。开发者需要大量不同编码格式、不同分辨率的视频测试源以验证播放器稳定性。VideoLink Navigator 分类整理多个公开测试资源，开发者可直接引用列表中的样本链接进行断网重连、码率切换、seek 性能等专项测试。

场景二：内容聚合站点运维。运维人员需要定期检查外部视频源或媒体资源域名的可访问性。项目提供的结构化域名列表可被集成至监控脚本，配合定时任务实现域名存活性的自动化巡检。

场景三：流媒体协议学习研究。研究人员在分析 HLS 与 DASH 的切片策略、加密机制或 CDN 调度行为时，需要大量真实案例支撑。本导航收录多个具有代表性的外链案例，可供学术级流量采样与行为分析使用。

场景四：影视资源整理工具开发。工具开发者需要获取分类准确的媒体域名模式以编写爬虫规则或去重逻辑。项目索引体系可作为规则种子库，辅助提升工具对媒体资源的识别准确率。

## 快速开始

以下操作适用于 Linux / macOS / Windows WSL 环境。项目本身不依赖运行时服务，仅需 Git 与基础 Shell 环境即可完成本地部署。

```bash
# 克隆仓库
git clone https://github.com/your-org/video-link-navigator.git
cd video-link-navigator

# 安装依赖（仅用于本地检测脚本，导航数据本身无需安装）
# 检测脚本使用 curl + jq，若未安装请先执行：
# Ubuntu/Debian: sudo apt install curl jq -y
# macOS: brew install curl jq
# Windows WSL: 同 Ubuntu 命令

chmod +x scripts/check_links.sh

# 运行链接有效性检测（示例）
./scripts/check_links.sh
```

## 贡献指南

欢迎社区贡献者提交新的视频链路、媒体域名或工具入口。请遵循以下流程确保索引质量与一致性。

第一步：查阅现有索引。在提交新链接前，请先检索 data/links_master.yaml 及 docs/link-spec.md，确认链接尚未收录，避免重复。

第二步：准备链接信息。使用 YAML 格式记录链接地址、分类标签、来源说明、协议类型及推荐检测方式。所有字段必须符合 link-spec.md 中定义的 Schema。

第三步：发起 Pull Request。将更新提交至社区审核分支，PR 描述中请明确说明链接用途、可用性验证结果以及预期归属分类。PR 标题建议使用 [ADD] / [REMOVE] / [UPDATE] 前缀。

第四步：等待审核与测试。项目维护者将运行自动化检测脚本验证链接真实可用性，并检查 YAML 格式合法性。审核通过后合并至主分支并更新 CHANGELOG。

第五步：关注反馈。合并后若检测脚本连续三次报告链接失效，该链接可能被移入 deprecated 列表，请贡献者及时确认或提供替代资源。

## 常见问题

Q：导航列表中的链接是否永久有效？

A：所有外部链接均由其所属第三方维护，本项目仅提供索引不保证永久可用。我们通过每日自动化检测记录每个链接的状态变化，并在 CHANGELOG 中标注失效日期。建议用户在使用前自行验证。

Q：我发现了失效链接，应该如何处理？

A：您可以选择提交 Issue 标注该链接及其失效日期，或直接提交 Pull Request 将其移入 deprecated_links.yaml。项目维护者会定期清理并更新主索引。如果同时有可替代的新链接，欢迎一并提交。

Q：能否将本项目集成到我的自动化工具中？

A：可以。data/links_master.yaml 采用结构化格式，支持被任何支持 YAML 解析的编程语言读取。您可以直接引用该文件作为外部数据源，无需额外 API 调用。引用时建议明确标注数据版本及获取日期，以便回溯。
