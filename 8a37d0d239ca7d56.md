# HyperLink Navigator

HyperLink Navigator 是一个面向开发人员、技术研究者与内容聚合者的轻量级外链资源导航系统。该项目不储存任何实际内容，仅提供结构化、可分类、可检索的互联网资源入口索引，帮助用户从分散的媒体源、流媒体站点、信息公告板与备用域名池中快速定位可用访问路径。

项目定位为“技术化的外链门禁系统”，核心目标用户包括自动化运维工程师、数据采集管道开发者、个人媒体库管理员以及需要频繁切换访问入口的海外内容观察员。系统通过静态索引表、可用性标记与分组策略，将大量不可预测的域名变动转化为可编程、可监控的稳定资源列表，从而降低人工记忆与手动切换的成本。

本项目不提供代理、不破解加密、不绕过访问限制，仅作为公开信息的整理与导航工具，所有收录链接均为互联网上可公开获取的地址，项目方不对链接内容的合法性与可用性作任何明示或暗示的保证。

## 功能概览

- **多级分类索引**：按媒体类型、地域、语言与更新频率将链接划分为不同类别，支持快速筛选与批量导出。
- **可用性标记系统**：允许用户为每个链接标注当前访问状态（有效、失效、需验证），并支持自定义标签与备注。
- **批量导入导出**：支持 CSV 与 JSON 格式的链接库备份与迁移，便于团队共享与版本管理。
- **本地静态部署**：完全基于静态 HTML 与 JavaScript，无需后端服务，可运行于任意 HTTP 服务器或本地文件系统。
- **命令行管理工具**：提供 Node.js 脚本用于链接去重、格式校验与健康检查（HTTP HEAD 请求）。
- **定时巡检机制**：可配合 cron 任务定期检测链接响应状态，自动生成可用性报告。
- **暗色主题与响应式布局**：界面适配桌面与移动设备，降低长时间使用的视觉疲劳。
- **URL 白名单与黑名单过滤**：支持正则表达式规则，辅助用户屏蔽广告域名或限定访问范围。

## 应用场景

- **个人媒体库辅助管理**：用户维护多个影视类域名收藏夹时，可借助 HyperLink Navigator 按主题（如动画、纪录片、综艺）分页管理，每次域名变动时仅需更新对应条目，无需修改书签栏。
- **数据采集管道入口配置**：爬虫开发者可将本项目的导出 JSON 作为上游配置源，每次启动采集任务前自动拉取最新链接列表，实现入口动态切换。
- **跨境内容观察与研究**：针对某些地区性内容服务频繁更换域名的情况，研究者可使用本项目记录历史域名与当前有效域名，形成时间轴追踪表，辅助分析域名轮换规律。
- **团队内部链接共享库**：小型团队可将本项目部署在内网 Git 仓库中，成员通过 Pull Request 方式提交新链接或更新状态，由维护者合并后统一发布，保持团队认知一致。
- **应急访问面板**：当主域名不可用时，用户可快速打开本项目页面，从备用域名列表中选择可用的替代入口，减少中断时间。

## 快速开始

以下步骤适用于 Node.js 环境（版本 16.x 及以上），建议在 Linux 或 macOS 终端中执行。

```bash
# 克隆仓库到本地
git clone https://github.com/your-org/hyperlink-navigator.git
cd hyperlink-navigator

# 安装依赖（仅需 npm）
npm install

# 启动本地开发服务器，默认监听 8080 端口
npm run serve
```

执行完毕后，在浏览器中访问 `http://127.0.0.1:8080` 即可看到导航面板首页。如需构建生产环境静态文件，请执行 `npm run build`，输出目录为 `./dist`。

## 贡献指南

1. 在 GitHub 上 Fork 本仓库，并克隆到本地开发环境，确保基于 `main` 分支创建新的功能分支（命名规则：`feature/描述` 或 `fix/描述`）。
2. 修改或新增链接条目时，请遵循 `data/sources.json` 中定义的 JSON Schema，务必包含 `url`、`category`、`status` 与 `lastChecked` 字段，并运行 `npm run validate` 校验格式。
3. 提交代码前执行 `npm run lint` 与 `npm run test` 确保代码风格与单元测试通过，并编写必要的变更说明文档（位于 `docs/changelog.md`）。
4. 发起 Pull Request 至 `main` 分支，描述中需清晰说明本次变更的目的、影响的链接列表或功能模块，并附上本地测试截图（如涉及 UI 变动）。
5. 等待维护者审核，审核通过后即可合并。合并后建议同步更新本地仓库并重新构建生产文件。

## 常见问题

**Q：某些链接无法访问，我应该如何处理？**

A：请首先确认本地网络环境是否能够正常解析该域名。如果确认网络正常但仍无法访问，您可以通过项目的“标记失效”功能将状态更新为“失效”，或直接在 `data/sources.json` 中将该条目的 `status` 字段修改为 `inactive`，然后提交 Pull Request。项目维护者会定期验证并移除长期失效的条目。

**Q：项目是否支持自定义添加非影视类链接？**

A：支持。HyperLink Navigator 的设计初衷是通用的链接索引工具，不限制类别。您可以在 `categories.json` 中添加新的分类定义，并在 `sources.json` 中为链接指定新的 `category` 字段值。前端面板会自动读取并显示新增的分类筛选项。

**Q：如何批量导入大量链接？**

A：推荐使用 CLI 工具中的 `import` 命令（需自行编写导入映射脚本）。您可以将 CSV 或 JSON 文件按照项目 Schema 转换后，使用 `node cli/import.js --file ./my-list.json` 导入。导入前请务必执行 `--dry-run` 参数进行预检查，避免格式错误导致数据损坏。
