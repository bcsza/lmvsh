# FilmLink Navigator

FilmLink Navigator 是一个面向数字媒体从业者、影视资源研究者及技术开发者的结构化外链导航系统。本项目不提供任何实体资源存储或分发服务，仅对互联网公开可访问的媒体相关域名进行逻辑分类与索引，帮助用户快速定位特定类型的影视数据源、流媒体测试节点或行业信息门户。

本项目定位为技术资源汇总工具，适用于需要批量管理外部媒体链接、构建私有影视资源索引或进行网络流媒体可用性测试的场景。通过标准化的目录结构与清晰的分类标签，用户可显著降低在分散网络中查找特定资源的时间成本。

## 功能概览

- **域名分类索引**：按媒体类型、地区、运营主体等维度对收录域名进行逻辑分组，支持多级标签筛选。
- **可用性探测**：内置 HTTP/HTTPS 头部检测模块，可批量验证每个域名的基础可达性与响应状态码。
- **自定义标签系统**：允许用户为每个条目添加自定义元标签（如 `4K`、`HDR`、`纪录片`、`动画`），便于个性化检索。
- **批量导入导出**：支持 CSV 与 JSON 格式的链接批量导入，以及过滤后的结果集导出。
- **黑白名单过滤**：基于正则表达式的域名级过滤规则，可屏蔽广告域名或仅显示白名单内资源。
- **定时刷新机制**：支持配置定时任务（cron 表达式），定期重新检测所有域名的可用状态并记录历史变化。
- **RESTful API 接口**：提供只读式 JSON API，允许第三方系统远程调用分类数据与检测结果。
- **静态站点生成**：可将当前索引数据渲染为纯静态 HTML 页面，适合部署到 CDN 或内部文档服务器。

## 应用场景

- **流媒体节点运维**：运维人员可使用本项目定期检测多个第三方流媒体备选域名的连通性，快速发现失效节点并切换路由。
- **影视数据采集预处理**：数据采集工程师可利用本项目的过滤与导出功能，从大量公开域名中筛选出符合特定地域或编码格式的候选源，作为采集任务的输入清单。
- **学术研究参考索引**：媒体研究学者可将本项目作为互联网影视资源分布现状的观察窗口，通过分类统计了解不同域名后缀、服务商的活跃比例。
- **个人媒体库构建辅助**：个人用户可将自己常用的影视查询、下载、字幕站点统一纳入本项目管理，配合标签功能实现高效的个人知识库检索。

## 快速开始

以下命令可在 Linux/macOS 或 Windows WSL 环境下完成本项目的初始部署与运行。确保系统已安装 Git 与 Node.js 18 及以上版本。

```bash
# 克隆项目仓库
git clone https://github.com/example/filmlink-navigator.git
cd filmlink-navigator

# 安装依赖（使用 npm）
npm install

# 运行初始索引构建与本地服务
npm run build
npm start
```

启动成功后，默认在本地 3000 端口提供 Web 管理界面与 API 服务。首次运行会自动生成示例数据并执行一次全量可用性检测。

## 常见问题

**Q：探测结果中显示部分域名为超时或拒绝连接，是否意味着这些资源永久不可用？**

A：不完全。超时或拒绝连接可能由临时网络波动、目标服务器防火墙策略、地域性访问限制等因素导致。建议调整探测超时时间（配置文件中的 `probe.timeout` 项）或在多个不同网络环境下多次执行检测。项目也支持配置代理服务器以绕过部分地域限制。

**Q：如何更新内置的域名列表？是否支持从远程 URL 自动拉取？**

A：本项目不强制锁定内置列表。您可以通过 `data/sources/` 目录下的 CSV 或 JSON 文件手动增删条目，然后执行 `npm run refresh` 命令重建索引。若需从远程 URL 拉取，可配置 `config/production.yaml` 中的 `remoteSource.url` 字段，调度器会在每次定时任务触发前自动下载并合并远程列表。

**Q：项目是否提供 GUI 图形界面？还是只能通过命令行操作？**

A：项目默认提供基于 Web 的管理界面，启动后通过浏览器访问 `http://localhost:3000` 即可使用。该界面支持域名搜索、标签筛选、手动触发探测以及结果导出等常用操作。API 接口则供自动化脚本或其他系统集成使用。
