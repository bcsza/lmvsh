# LinkVault

LinkVault 是一个轻量级的技术资源导航与外链聚合工具，定位于个人开发者、技术团队及内容创作者的技术收藏与分享场景。该项目并非传统的网址导航站，而是以工程化方式管理高质量外部资源链接，提供分类展示、状态监控、访问统计与快速检索能力，适用于自部署的技术文档站、内部开发工具集推荐页或个人品牌的知识库入口。

LinkVault 面向以下用户群体：需要统一管理大量技术文档链接的架构师、维护团队内部常用工具索引的 DevOps 工程师、希望以简洁优雅方式分享优质资源的博主，以及需要快速搭建项目外链聚合页的开源爱好者。项目核心目标是解决资源链接分散、失效难以追踪、缺乏分类检索等痛点，帮助用户将散落在浏览器书签、临时笔记、聊天记录中的高价值 URL 集中整合为结构化、可维护、可公开访问的资源清单。

## 功能概览

- **多级分类管理**：支持为每个链接分配分类标签与子分组，内置前端框架、后端服务、运维工具、学习资料、视频资源等默认分类体系，用户可自定义增删改分类结构。

- **链接状态监测**：后台定时任务自动检测所有外链的可访问性，标记失效链接并记录响应时间变化趋势，便于运维人员及时清理或更新已失效的资源地址。

- **全文检索与过滤**：基于标题、描述、分类标签、域名等多维度字段提供快速关键词检索，同时支持按分类、按状态（有效/失效）、按访问热度进行筛选排序。

- **简洁展示模式**：提供列表视图与卡片视图两种展示风格，卡片视图自动提取网站 Favicon 与页面标题，列表视图紧凑展示链接核心属性，适配不同浏览偏好。

- **访问统计看板**：记录每个外链的点击次数与最近访问时间，生成热门资源排行榜，帮助识别高频使用的核心工具与文档。

- **数据导入导出**：支持批量导入浏览器书签 HTML 文件、JSON 格式链接数据集，同时支持将当前链接库导出为 JSON、Markdown 或 CSV 格式，方便迁移与备份。

- **自定义主题与布局**：内置浅色/深色主题切换，允许用户自定义页面标题、Logo、页脚信息以及首页展示的推荐链接数量。

## 应用场景

- **团队内部技术栈索引**：开发团队可将常用中间件文档、监控面板地址、CI/CD 流水线入口、代码仓库链接统一收录至 LinkVault 部署实例，新成员入职时无需逐个询问即可快速获取所有必需工具入口。

- **开源项目附属资源导航**：开源项目维护者可在项目文档中嵌入 LinkVault 外链页，统一整理相关生态工具、社区论坛、视频教程、扩展组件等外部资源，降低用户寻找辅助资料的成本。

- **个人知识库外部引用管理**：技术博主或知识管理爱好者可将其博客、Wiki 或数字花园中频繁引用的外部文章、视频、官方文档集中托管于 LinkVault，在写作时仅需引用固定锚点，避免链接散落导致维护困难。

- **技术活动或课程配套资源页**：技术会议、线上训练营或大学课程讲师可使用 LinkVault 快速生成配套资源手册，集中放置课件链接、实验环境入口、参考阅读列表，活动结束后仍可长期保留供学员回顾。

## 快速开始

以下命令演示了从代码仓库克隆、安装依赖到启动开发服务器的完整流程，适用于本地测试与二次开发。

```bash
# 克隆代码仓库
git clone https://github.com/your-org/linkvault.git
cd linkvault

# 安装项目依赖（使用 pnpm 或 npm）
pnpm install
# 若未安装 pnpm，可使用 npm install

# 复制环境变量模板并填写必要配置
cp .env.example .env.local

# 启动开发服务器（默认监听 3000 端口）
pnpm dev
# 或 npm run dev
```

生产环境部署建议先执行构建命令 `pnpm build`，再使用 `pnpm start` 启动生产服务器，并配合 Nginx 或 Caddy 进行反向代理。

## 常见问题

**Q：部署后首次访问页面空白或报错，如何排查？**

A：请按以下顺序依次检查：首先确认 `.env.local` 文件中数据库连接字符串（DATABASE_URL）和 Redis 连接地址（REDIS_URL）是否正确，且对应服务已启动并可访问。其次运行 `pnpm db:push` 执行数据库 Schema 同步，确保数据表已创建。最后查看终端日志输出，若有具体错误堆栈可参照 /docs/deployment 中常见错误码对照表进行定位。

**Q：链接状态监测显示大量失效，但手动访问浏览器中正常，是什么原因？**

A：监测服务默认使用无头浏览器发起 HTTP 请求，部分网站可能对无头模式或非浏览器 User-Agent 返回 403 或 429 状态码。您可在环境变量中调整 `CHECKER_USER_AGENT` 为真实浏览器值，或增加 `CHECKER_REQUEST_TIMEOUT` 超时时间。同时检查监测服务所在网络环境是否需要配置代理。

**Q：如何将现有浏览器书签一次性导入 LinkVault？**

A：在任意浏览器中导出书签为 HTML 文件（通常为 Netscape Bookmark 格式），然后登录 LinkVault 管理后台，在「数据管理」-「导入」模块中上传该 HTML 文件，系统将自动解析书签标题、URL 及文件夹层级，并映射至 LinkVault 的分类体系。导入完成后可在「未分类」视图中手动调整部分链接的分类归属。
