# WebNavigator Hub

WebNavigator Hub 是一个面向开发者、技术研究人员与信息分析人员的轻量级导航与资源聚合工具。该项目不提供具体业务功能，而是作为互联网公开资源的结构化索引与访问入口，帮助用户以可复现、可扩展的方式组织和管理高频访问的外部链接。

项目定位为“技术人员的浏览器起点页”，解决以下核心问题：日常开发中需要频繁访问大量外部工具、文档、社区与数据源，但浏览器书签管理混乱、跨设备同步不便、缺乏版本控制与团队共享能力。WebNavigator Hub 通过纯静态配置与本地优先的架构，将资源列表以代码形式托管，支持 Git 版本管理、私有化部署与快捷自定义。

目标用户包括：运维工程师、全栈开发者、网络安全研究人员、技术写作人员以及任何需要系统化管理外链资源的专业人士。

## 功能概览

- **多级分类导航**：支持按技术领域、使用频率、来源批次等多维度对链接进行分组，便于快速定位目标资源。

- **纯静态配置**：所有链接数据存储于单一 JSON 或 YAML 文件中，无需数据库，开箱即用，便于脚本化处理与持续集成。

- **本地优先启动**：项目提供轻量级 HTTP 服务，默认监听本地端口，启动后即可在浏览器中访问导航面板，无外部依赖。

- **一键快速添加**：通过命令行交互脚本，用户可输入名称、URL、分类与标签，系统自动校验 URL 格式并更新配置文件。

- **链接可用性检查**：内置基于 HEAD 请求的存活检测模块，可定时或手动触发，输出不可用链接报告，辅助资源维护。

- **深色/浅色主题切换**：前端界面适配系统主题偏好，同时提供手动切换按钮，提升长时间浏览体验。

- **导出与分享**：支持将当前配置导出为纯文本列表、Markdown 表格或 HTML 书签文件，便于离线使用或团队分发。

- **Docker 交付**：提供官方 Docker 镜像，支持一键容器化部署，适用于云服务器或内部集群环境。

## 应用场景

- **个人开发环境初始化**：新购云主机或重装系统后，开发者可通过 Git 克隆本项目，一键启动导航页，快速访问常用镜像仓库、文档站与监控面板，省去手动搜索和输入 URL 的步骤。

- **团队技术文档入口**：技术团队可将本项目部署为内部知识库的统一入口，集中存放 Confluence、Jira、GitLab、SonarQube 等内部系统的链接，新人入职时即可获得完整访问路径。

- **安全研究信息汇集**：安全研究人员可利用分类标签管理不同威胁情报源、CVE 数据库、沙箱分析平台和漏洞披露站点，配合可用性检查功能，及时移除失效情报源。

- **多项目环境切换**：运维人员可通过复制配置文件并修改分类前缀，快速生成针对不同客户环境（如生产、预发布、测试）的独立导航实例，避免跨环境访问混淆。

- **技术写作资料库**：技术博主或文档工程师可集中收藏各类规范文档、设计参考、API 示范代码库和在线编辑器，在写作过程中快速查阅，提升内容产出效率。

## 快速开始

以下步骤适用于 Linux / macOS / Windows WSL 环境。

```bash
# 克隆仓库
git clone https://github.com/your-org/webnavigator-hub.git
cd webnavigator-hub

# 安装依赖（使用 Python 3.9+）
pip install -r requirements.txt

# 运行本地服务（默认监听 127.0.0.1:8080）
python server.py
```

启动后，在浏览器中访问 `http://127.0.0.1:8080` 即可看到导航面板。配置文件位于 `data/links.yaml`，可直接编辑或通过 `cli.py` 交互式添加。

| Docker Engine 20.10+ | 否 | 仅容器化部署时需要，支持 Docker Compose 可选 |
| Git 2.25+ | 否 | 仅通过源码更新或贡献代码时需要，普通运行无需 |

## 常见问题

**Q: 为什么访问某些链接时提示“连接被拒绝”或“超时”？**  
A: 本项目仅作为导航索引，不对外部资源的可用性负责。您可以使用内置的 `python cli.py check --all` 命令批量检测当前配置中所有链接的响应状态。若某个链接持续不可达，建议通过 `cli.py remove` 移除或联系对应站点管理员。网络环境（如防火墙、代理）也可能影响检测结果。

**Q: 如何将我的自定义配置迁移到另一台机器上？**  
A: 只需复制 `data/links.yaml` 文件到新机器的相同相对路径下，并确保项目版本一致。若使用 Docker，可通过挂载卷方式将配置文件映射到容器内 `/app/data/links.yaml`。不推荐直接备份整个项目目录，避免携带平台特定的缓存或日志文件。

**Q: 该项目是否支持多用户权限管理或登录功能？**  
A: 不支持。WebNavigator Hub 定位于本地优先、单用户或信任团队内部使用的工具，不提供身份认证与授权机制。若需要多用户隔离，建议为每个用户独立部署一份实例，或在前端使用反向代理做基础访问控制（如 HTTP Basic Auth）。
