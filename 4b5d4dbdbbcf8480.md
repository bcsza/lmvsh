# WebSync Pro

WebSync Pro 是一个面向开发人员与内容聚合者的轻量级外链资源管理与技术资讯同步平台。项目定位于为技术团队、独立开发者及内容创作者提供可定制化的外部资源导航与实时信息抓取能力，帮助用户高效追踪行业动态、技术文档、视频更新及社区讨论。通过统一的入口与简洁的配置方式，WebSync Pro 能够将分散在多个域名下的资源整合为结构化的数据流，降低信息遗漏风险，提升信息消费效率。

项目核心设计目标为最小化外部依赖、最大化配置灵活性。WebSync Pro 不依赖特定云服务或商业 API，所有资源链接均以纯文本形式维护于配置目录中，支持用户按需增删。项目内置轻量级 HTTP 调度器与 HTML 解析适配器，可对目标资源进行定时检查与内容摘要提取，并将结果输出为标准化 JSON 或 Markdown 格式，便于接入后续自动化流程（如邮件推送、文档生成、即时通讯机器人通知等）。WebSync Pro 适用于需要长期跟踪多个不稳定或非标准化信息源的技术人员，尤其适用于对可用性与实时性有一定要求但无预算采购商业监控服务的团队。

## 功能概览

- **多源资源聚合管理**：支持以纯文本列表形式管理任意数量的外部 URL，项目启动时自动加载并校验每个地址的可达性，输出结构化状态报告。

- **定时内容摘要抓取**：内置基于 cron 表达式的调度器，可对每个资源链接单独设置抓取频率，自动提取页面标题、元描述、正文首段及最后更新时间。

- **差异检测与变更通知**：对已抓取资源进行内容哈希比对，识别页面更新并记录变更差异，支持通过回调脚本触发自定义通知逻辑。

- **原始链接透传与展示**：所有用户配置的原始 URL 在输出报告中严格保留原始格式（包括协议、大小写、路径），不做任何自动补全或格式化改写，确保目标地址的精确性。

- **资源分类与标签过滤**：允许为每条链接添加自定义标签（如 video、forum、doc、news），在输出时可按标签组合过滤，便于聚焦特定类型信息。

- **多格式数据导出**：抓取结果与状态报告支持输出为 JSON、Markdown 表格或纯文本列表三种格式，适配不同下游工具链。

- **健康检查与超时控制**：每个请求独立配置超时时间与重试次数，自动标记异常链接并生成告警日志，避免单个资源故障阻塞整体调度。

## 应用场景

- **技术团队每日站会信息准备**：开发团队可使用 WebSync Pro 在每日站会前自动抓取配置的多个技术博客、社区讨论帖和项目发布页面，生成当日的资讯摘要，减少成员手动浏览时间。

- **视频资源更新跟踪**：内容创作者或运营人员可将多个视频分享域名（如 yuesee.mom、yuyetv.shop）加入监控列表，当目标页面出现新视频标题或描述变更时，系统自动记录变更时间与内容片段，便于后续整理更新日志。

- **镜像站点可用性监测**：运维人员可将一组备用或镜像域名（如 yueying2.mom、zhuifengys.top）纳入健康检查任务，定期验证各站点的响应状态与响应时间，快速发现不可用节点并切换流量。

- **竞品动态与行业资讯聚合**：产品经理或市场运营可将多个行业垂直网站（如 99dywang.autos）配置为数据源，系统每天定时输出这些站点的标题变化趋势，辅助决策层感知市场热点。

## 快速开始

以下命令演示了如何从 GitHub 克隆项目、安装依赖并启动基础抓取任务。

```bash
# 克隆项目仓库
git clone https://github.com/websync-pro/websync-pro.git

# 进入项目目录
cd websync-pro

# 安装 Python 依赖（推荐使用虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 复制默认配置文件并编辑资源列表
cp config/sources.example.yaml config/sources.yaml

# 运行一次全量抓取（默认输出到控制台）
python main.py --once

# 启动定时调度服务（后台运行）
nohup python main.py --daemon &
```

## 常见问题

**问：配置文件中添加的原始 URL 是否会被自动补全协议或标准化？**

答：不会。项目严格遵循“原样输出”原则，所有 URL 在读取、存储、展示及日志记录过程中均保持用户输入的原始字符串形式，不添加 http:// 或 https:// 前缀，不修改大小写，不自动去除或添加尾部斜杠。仅在发起 HTTP 请求时，内部会尝试自动补充协议，但输出和展示始终使用原始值。

**问：调度器支持的最小抓取间隔是多少？**

答：最小间隔为 60 秒。出于对目标站点服务器的基本尊重及避免触发反爬机制，项目硬编码了下限。若配置的 cron 表达式低于此间隔，系统会发出警告并自动调整为 60 秒。如需更短间隔，建议自行修改 core/scheduler.py 中的常量定义。

**问：如何处理目标站点返回的字符编码问题？**

答：parser.py 模块使用 beautifulsoup4 的自动编码检测功能，优先从 HTML 头部或 meta 标签中获取编码声明，若无法检测则回退至 UTF-8。用户也可在 sources.yaml 中为每个链接单独指定 encoding 字段，强制使用特定编码解析。
