# TechLink Navigator

TechLink Navigator 是一个面向开发人员与技术研究者的高质量外部资源导航与聚合系统。该项目不存储、不托管任何实体内容，仅以目录与链接形式对互联网公开的技术资源进行结构化整理与分类，旨在解决信息分散、检索效率低下的问题。项目定位为“技术外链的精选索引”，适用于日常开发查阅、技术调研、数据采集源梳理等场景。

目标用户包括后端工程师、数据工程师、运维人员、安全研究人员以及技术团队的管理者。通过明确的分类体系和逐条人工校验的链接状态，本项目帮助用户在繁杂的网络信息中快速定位可用、稳定且内容垂直的外部站点，从而提升信息获取效率。

## 功能概览

- 智能分类索引：按资源类型、内容主题、使用频次等多维度划分，每个外链附带简要说明标签。
- 实时状态标记：定期对收录链接进行可访问性检测，并在目录中标注可用性状态。
- 多级筛选视图：支持按技术栈（如 Python、JavaScript、DevOps）、资源性质（文档、工具、视频、数据源）快速过滤。
- 版本化更新记录：每次增删改链接均记录变更日志，确保资源库可追溯。
- 用户自定义收藏夹：允许用户在本地克隆后维护个人常用子表，不影响主库结构。
- 批量导出功能：支持将选中的链接集合导出为 JSON、CSV 或纯文本格式，便于二次处理。
- 黑暗模式与阅读优化：前端预览模板内置高对比度主题，提升长时段浏览舒适度。

## 应用场景

1. 技术调研阶段快速筛选数据源：当开发团队需要接入第三方 API 或采集公开数据时，可通过本导航直接定位到内容垂直、更新频繁的站点，减少盲目搜索时间。

2. 运维监控补充外部参考源：运维人员在配置健康检查或异常报警规则时，可引用导航中的可用性状态标记，辅助判断是目标站点问题还是自身网络环境问题。

3. 新人入职环境搭建参考：新成员加入团队后，可通过本导航快速了解项目依赖的外部资源分布，包括文档站、示例代码库、社区论坛等，加速上手进程。

4. 爬虫开发维护规则库：数据采集工程师可将导航中的链接作为种子 URL 库，结合状态标记动态调整抓取策略，降低维护成本。

## 快速开始

以下命令请在终端中依次执行，默认使用 HTTPS 协议克隆。若网络受限，可改用 SSH 方式。

```bash
git clone https://github.com/techlink-navigator/navigator.git
cd navigator
npm install
npm run build
```

执行完毕后，`dist` 目录下将生成可部署的静态文件。若仅需浏览导航内容，可直接打开 `index.html` 预览。

## 贡献指南

1. Fork 本仓库并创建新分支，分支命名建议为 `feat/add-source-<category>` 或 `fix/update-status-<date>`。

2. 在 `data/sources/` 下对应分类的 JSON 文件中新增链接条目，必须包含 `url`、`title`、`description` 与 `tags` 字段，并确保 `url` 字段值与原始用户提供的一致，不做任何改写。

3. 提交前运行 `npm run validate` 校验数据格式，确保所有新增条目通过 schema 检查。

4. 若需调整链接状态（如标记为失效），请修改 `data/status/availability.cache` 对应条目，并在提交信息中注明检测时间与依据。

5. 发起 Pull Request 至主仓库的 `main` 分支，描述变更内容与测试情况，等待维护者审核。

## 常见问题

Q: 为什么某些链接在导航中显示为失效，但我手动访问却是正常的？
A: 状态检测基于服务端网络环境与超时设置，若您所在的网络与检测源存在差异，可能导致结果不一致。您可以手动修改本地缓存文件中的状态标记，或向维护者提交最新的可用性反馈。

Q: 我能否将本导航完全离线使用？
A: 可以。克隆仓库并构建后，所有链接数据均为静态 JSON 文件，您无需任何网络即可浏览目录。但状态检测功能与外部资源跳转自然需要网络支持。

Q: 如何请求新增一个分类或链接？
A: 请按贡献指南提交 PR，或在 Issues 板块中使用 “Resource Request” 模板提交需求，维护者会评估分类合理性后合并。
