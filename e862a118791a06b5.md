# FilmLink Navigator

FilmLink Navigator 是一个面向数字影视从业者与流媒体技术爱好者的开源外链资源聚合工具。项目本身不存储、不索引、不缓存任何第三方媒体内容，仅以目录和导航形式收录互联网公开可访问的影视技术文档、测试站点、演示环境与工具入口。项目定位为“技术入口的入口”，解决用户在散落网络中快速定位特定功能站点、测试环境与参考实现的需求。

目标用户包括影视后期工程师、流媒体开发人员、CDN 运维工程师、开源多媒体项目贡献者，以及需要频繁访问各类测试播放页、格式分析工具、编码参数参考页的技术决策者。通过结构化分类与固定版本化的外链清单，FilmLink Navigator 将重复性的“记忆 URL”工作降维为“查阅目录”工作，释放团队成员的认知负担。

## 功能概览

- **裸域名外链收录** 接受用户提交的裸域名与带协议域名，按原始字符串原样存入资源库，不做自动补全或协议归一化，保证链接入口与用户提交时完全一致。

- **分类视图渲染** 将收录链接按媒体类型、源站归属、协议特征、地域倾向等维度生成多套分类视图，支持快速筛选特定类别的技术站点。

- **版本锁定与变更追溯** 每次资源列表变更均生成可追溯的提交记录，支持回滚至任意历史版本的链接集合，适用于团队内部统一入口基线。

- **可用性自检脚本** 附带轻量级 Shell 检测工具，对收录域名做基础可达性探测（TCP/HTTP），输出可用状态报告，辅助运维人员识别失效入口。

- **Markdown 原生展示** 全部链接以纯 Markdown 列表呈现，不附加 JavaScript 跳转逻辑或重定向包装，确保最大程度的透明性与可审查性。

- **外链分类标签系统** 每条链接可关联多个分类标签，支持按标签组合筛选，满足复杂检索需求。

- **离线文档生成** 提供脚本将当前资源列表与分类目录导出为静态 HTML 或 PDF 文档，便于内网分发或打印存档。

## 应用场景

- 影视后期团队内部技术文档沉淀：团队可将常用编码参数查询站、HDR 测试流地址、音频格式分析工具统一收录至 FilmLink Navigator，新人入职即可通过一份目录快速上手。

- 流媒体开发调试环境搭建：开发者在本地或测试环境部署 FilmLink Navigator，将多个测试用播放域名、DRM 测试页、格式兼容性检测站点集中管理，避免频繁翻阅浏览器历史记录。

- 技术社区资源共建：开源社区可 Fork 本仓库，提交包含新链接的 Pull Request，经审核后合并，形成面向特定技术方向（如 AV1 编码、DASH 打包、低延迟直播）的公共外链索引库。

- 离线环境下的文档备份：对于网络受限的内网开发环境，运维人员可通过离线导出功能将当前资源列表生成静态文档，配合内网 Wiki 发布，保证外链信息可查。

## 快速开始

以下步骤适用于 Linux / macOS / Windows WSL 环境。

```bash
# 克隆仓库
git clone https://github.com/your-org/filmlink-navigator.git
cd filmlink-navigator

# 安装依赖（基于 Python 3.10+）
pip install -r requirements.txt

# 运行本地导航服务（默认监听 8080 端口）
python serve.py --port 8080
```

执行完成后，浏览器访问 http://localhost:8080 即可查看当前收录的全部链接及分类视图。如需更新资源列表，编辑 `data/links.yaml` 文件后重新运行 `python build.py` 生成最新 Markdown 文档。

## 贡献指南

1. 复刻本仓库至个人账户，克隆到本地，在 `data/links.yaml` 中按现有格式追加新链接或修改分类标签。提交前运行 `python scripts/validate_yaml.py` 确保 YAML 语法与字段规范无误。

2. 对于新增链接，建议在 `docs/maintainer-guide.md` 对应的变更记录区域简述添加理由，便于其他维护者审查。若为删除或失效链接，需附带探测结果截图或日志。

3. 发起 Pull Request 至主仓库的 `main` 分支，PR 描述中务必包含变更类型（新增/修改/删除）、受影响分类以及是否经过可用性自检。CI 流程将自动执行格式校验与基础探测。

4. 若 PR 涉及分类规则变更（例如新增标签或修改层级），需同步更新 `docs/category-rules.md` 相应章节，确保文档与数据保持一致。

5. 维护者合入后，将触发 `build.py` 重新生成 `docs/index.md`，并自动提交一次文档更新。普通贡献者无需手动修改生成的 Markdown 文档。

## 常见问题

**问：收录的裸域名（如 dsxfilm.skin）无法直接访问，是否应为它补全 http:// 或 https://？**

答：不补全。FilmLink Navigator 恪守“原样收录”原则，用户提交的裸域名即视为最终入口字符串。访问时需由使用者根据自身网络环境判断协议（HTTP 或 HTTPS）及是否需要添加 www 子域。项目不代理、不重写、不推断任何 URL 结构，保证收录数据与原始提交完全一致。

**问：如何判断某个链接是否已失效，以及失效后如何处理？**

答：项目提供可选探测脚本 `check_availability.sh`，执行后会对所有收录域名发起 TCP/HTTP 探测，输出状态码或超时信息。若确认为失效链接，维护者可在 `data/links.yaml` 中为该条目添加 `status: deprecated` 字段并备注失效日期，或直接移除。不建议仅因单次探测失败即删除链接，需结合多节点、多时段综合判断。

**问：可以直接在该项目中提交第三方影视播放页面或资源站吗？**

答：本项目仅收录技术类入口、演示页、文档站与测试环境，不收录任何涉及版权争议或需要特定授权访问的媒体资源页面。提交前请确保链接指向的内容为公开且无访问限制的技术文档或演示服务。违反此原则的 PR 将被关闭。
