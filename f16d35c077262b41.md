# ResourceForge

ResourceForge 是一个面向技术团队与独立开发者的外链资源聚合与导航工具，致力于解决“优质技术资源分散、检索成本高、版本信息混乱”的普遍痛点。项目本身不存储任何实体文件或数据，而是以结构化索引方式，将大量经过人工筛选的技术文档、社区入口、数据集、开发工具及基础设施地址集中管理，并通过清晰的项目结构与标准化文档输出，帮助用户快速定位目标资源，降低信息查找与交接成本。

ResourceForge 适用于需要频繁查阅外部技术资料的技术负责人、运维工程师、架构师以及开源贡献者。项目采用纯静态 Markdown 与脚本化生成方式，易于二次开发，可无缝嵌入 CI/CD 流程，作为团队内部知识库的统一导航层。

## 功能概览

- **多级资源索引**：按技术领域、适用场景、数据格式等多个维度对资源链接进行分类，支持深度标签嵌套与组合筛选。
- **自动链接可用性检测**：集成轻量级健康检查脚本，可定时探测每条外链的可达性与响应状态码，自动标记异常条目。
- **结构化元数据标注**：每条资源均附带来源、更新日期、所属组织、协议类型等元信息，便于审计与版本追溯。
- **批量导入与导出**：支持从 CSV、JSON 及标准书签文件批量导入链接数据，亦可一键导出为 HTML 书签页或 RSS 订阅源。
- **版本化快照记录**：每次资源列表更新均生成变更日志（CHANGELOG），支持回溯任意历史版本的资源集合。
- **自定义视图模板**：提供多种文档渲染模板（表格、卡片、列表、时间线），用户可根据使用习惯切换展示风格。
- **CLI 管理工具**：提供命令行交互接口，支持资源增删、标签重命名、批量校验等运维操作，无需图形界面。
- **团队协作锁机制**：基于文件锁与操作日志，防止多人同时编辑导致的冲突，适用于集中式知识库维护场景。

## 应用场景

- **新员工技术入职导航**：将团队常用的云平台控制台、监控面板、日志系统、代码仓库、CI/CD 入口等链接统一收录，新成员克隆项目后即可一键访问全部基础设施，显著缩短环境熟悉周期。
- **开源项目外部依赖索引**：开源维护者可将项目所依赖的第三方库文档、API 参考、数据样本源、社区论坛等外链集中管理，便于贡献者快速理解项目生态全貌。
- **技术调研与竞品分析**：在进行技术选型或竞品分析时，可将大量待评估产品的官网、技术白皮书、GitHub 仓库、性能对比报告等链接分类归档，并利用健康检查功能定期确认文档有效性，避免调研过程中出现链接失效。
- **离线文档辅助导航**：对于需要在内网隔离环境工作的团队，可将外链列表与本地镜像文档地址混合编排，构建统一的内部导航门户，兼顾安全性与易用性。

## 快速开始

以下步骤适用于 Linux / macOS / Windows WSL 环境，确保已安装 Git 与 Python 3.8 及以上版本。

```bash
# 克隆项目仓库
git clone https://github.com/resourceforge/resourceforge.git
cd resourceforge

# 安装依赖（建议使用虚拟环境）
python3 -m venv venv
source venv/bin/activate  # Windows 下使用 venv\Scripts\activate
pip install -r requirements.txt

# 初始化资源数据库并运行本地预览服务
python forge.py init
python forge.py serve --port 8080
```

启动后，在浏览器中访问 http://localhost:8080 即可查看资源导航主页。首次运行将自动生成示例资源分类与占位数据，用户可依据实际需求进行增删改。

## 贡献指南

我们欢迎并鼓励社区贡献，无论是新增资源链接、改进文档、修复缺陷还是提议新特性。请遵循以下步骤参与项目：

1. 在 GitHub 上 Fork 本仓库，并克隆到本地开发环境。确保你的分支基于最新的 main 分支创建。
2. 对于新增资源或修改现有索引，请统一编辑 `data/resources.json` 文件，并严格遵守 `docs/metadata-schema.md` 中定义的字段结构与取值范围。提交前请运行 `python forge.py validate` 进行格式校验。
3. 若涉及代码变更（CLI 命令、核心逻辑或模板），请同步补充对应的单元测试（位于 `tests/` 目录），并确保所有现有测试用例通过。执行 `pytest` 以验证。
4. 提交 Pull Request 时，请清晰描述变更目的、影响范围以及测试结果。若为新增功能，建议附上使用示例或截图。
5. 重大变更或架构调整建议先通过 Issue 进行讨论，达成初步共识后再投入开发，避免无效工作。

## 常见问题

**Q：健康检查脚本提示“连接超时”，但浏览器可以正常访问该链接，是什么原因？**

A：默认健康检查采用 curl 的 `--connect-timeout` 和 `--max-time` 参数，若目标服务器对非浏览器 User-Agent 或特定 IP 段有限制策略，可能导致超时。建议在 `config.yaml` 中调整 `check_timeout` 值（单位秒），或修改 `scripts/daily-check.sh` 中的 User-Agent 伪装选项。此外，部分内网服务可能需要配置代理，请检查环境变量 `HTTP_PROXY` 与 `HTTPS_PROXY`。

**Q：如何将现有浏览器书签批量导入 ResourceForge？**

A：项目支持 Netscape 书签格式（即大多数浏览器导出的 .html 书签文件）。使用命令 `python forge.py import --type bookmarks --file bookmarks.html` 即可自动解析并导入。导入前建议先执行 `--dry-run` 参数进行预览，确认分类映射无误后再实际执行。若需自定义标签映射规则，可参考 `docs/usage/import-export.md` 中的高级配置说明。

**Q：资源列表中的链接发生变化（如域名迁移或路径调整），如何批量更新？**

A：推荐使用 `python forge.py check --update` 命令，该命令会重新校验所有链接，并允许用户在交互式界面中为每个失效链接输入新的 URL 或标记为废弃。对于已知的批量替换场景（如旧域名替换为新域名），可使用 `python forge.py replace --old old-domain.com --new new-domain.com` 进行字符串替换操作，但需谨慎确认替换范围，避免误改。
