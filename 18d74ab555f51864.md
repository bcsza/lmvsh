# HyperLink Nexus

HyperLink Nexus 是一个面向技术内容创作者、开源项目维护者以及数字资源管理者的高可靠性外链汇聚与分发平台。本项目并非传统的内容管理系统，而是一个轻量化、可自托管的导航枢纽，专注于解决技术社区中优质外部资源分散、链接失效、入口不统一等问题。通过结构化的数据组织与简洁的交互界面，HyperLink Nexus 帮助用户将散落在各个域名的技术文档、工具站点、社区论坛以及多媒体资源整合为统一的访问入口，从而降低信息获取成本，提升团队协作与文档引用的效率。

本项目特别适用于需要维护大量外链的技术博客体系、开源项目的官方文档站、企业内部技术中台的知识库导航，以及个人开发者用于聚合学习资料的信息门户。HyperLink Nexus 本身不存储任何第三方内容，仅提供链接的元数据管理与渲染能力，确保合规性与轻量级部署。通过本平台，用户可以快速创建按领域、标签或用途分类的链接集合，并生成可嵌入现有网站的外链模块，实现真正的“一处维护，多处引用”。

## 功能概览

**批量链接导入与去重** 支持从 CSV、JSON 及 OPML 格式批量导入现有链接库，并自动检测重复 URL，保留最新更新的条目，避免人工整理的冗余劳动。

**智能链接状态监测** 内置周期性链接可用性检查机制，自动标记返回 4xx 或 5xx 状态码的失效链接，并生成报表供管理员审阅，有效维护链接库的健康度。

**自定义分类树与标签系统** 允许创建无限层级的分类目录，并为每条链接附加多个自定义标签，支持按分类、标签或关键词组合进行快速筛选与检索。

**元数据增强与备注管理** 每条链接可存储标题、描述、作者、发布日期、关联项目等扩展元数据，并支持内部备注字段，便于团队内部沟通链接的使用意图与注意事项。

**公开 API 与嵌入小部件** 提供 RESTful API 接口，允许第三方系统查询链接数据；同时生成可自适应的 JavaScript 小部件代码，将指定分类的链接列表嵌入任何 HTML 页面。

**用户权限与审核工作流** 支持多用户角色（管理员、编辑者、访客），编辑者提交的新链接或修改需经管理员审核方可对外发布，确保外链质量与安全性。

## 应用场景

技术博客与知识库的参考文献管理：技术作者在撰写文章时，通常需要引用大量外部规范、工具仓库或社区讨论。HyperLink Nexus 可作为独立的参考文献索引库，为每篇文章生成专属的链接集合，并通过 API 自动生成文章尾部的参考链接列表，避免手动维护的错漏。

开源项目官方文档的外链整合：开源项目往往依赖多个第三方服务（如 CI/CD 平台、代码覆盖率工具、容器镜像仓库）。项目维护者可使用 HyperLink Nexus 创建“生态工具链”页面，统一列出所有相关服务的入口与简要说明，大幅提升新贡献者的上手体验。

企业内部技术中台的导航门户：大型企业内部的研发体系涉及众多内部系统（代码库、制品库、监控面板、日志平台）。通过 HyperLink Nexus 搭建内部导航，配合权限控制，可确保团队成员快速定位所需资源，同时隔离敏感链接的访问范围。

个人开发者的学习资源聚合：自学新技术时，开发者常收藏数十个教程、视频、在线编辑器等链接。利用 HyperLink Nexus 的分类与标签功能，可以构建个人专属的学习路径导航，并随时同步至多台设备，实现书签的跨浏览器与跨设备统一管理。

## 快速开始

以下命令适用于 Linux / macOS / Windows WSL 环境，请确保已安装 Git 与 Node.js（v18 或以上版本）。

```bash
# 克隆项目仓库
git clone https://github.com/hyperlink-nexus/hyperlink-nexus.git

# 进入项目目录
cd hyperlink-nexus

# 安装所有依赖（使用 npm）
npm install

# 复制示例环境变量文件并编辑数据库连接等配置
cp .env.example .env

# 执行数据库迁移与初始数据填充
npm run migrate
npm run seed

# 以开发模式启动应用（默认监听 http://localhost:3000）
npm run dev
```

启动成功后，访问终端输出的本地地址即可进入管理面板。首次使用需按照提示创建管理员账户。

## 常见问题

Q: 项目是否支持 SQLite 或 MySQL 作为替代数据库？

A: 目前 HyperLink Nexus 的数据库抽象层主要针对 PostgreSQL 进行优化，包括 JSONB 字段操作与部分高级索引。理论上可通过调整 Sequelize 配置连接 MySQL，但部分查询语法可能存在兼容性问题，且未经过充分测试。我们强烈建议使用 PostgreSQL 以获得最佳性能与稳定性。

Q: 链接状态监测的频率是多少？是否会消耗大量资源？

A: 监测服务默认每 6 小时执行一次全量检查，并采用随机延迟分布以避免对目标服务器造成突发压力。每次检查仅发送 HEAD 请求，并设置 5 秒超时，单次检查对网络与 CPU 的消耗极低。对于大型链接库（超过 5000 条），建议通过环境变量调整监测批次大小与间隔时间。

Q: 如何从旧版导航系统迁移数据到 HyperLink Nexus？

A: 我们提供了从主流书签管理器（如浏览器 HTML 导出、Raindrop.io、Pocket）的转换脚本，位于 `scripts/migrate/` 目录下。您只需将导出的文件放置于指定目录，并执行 `npm run migrate:import` 即可自动完成格式转换与导入。对于自定义格式，可参考导入服务文档编写简单的映射配置。
