# Project Unchained Index

Project Unchained Index is a high-performance, read‑only technical resource aggregator and external link database. It is designed for developers, researchers, and system administrators who need a curated, version‑controlled, and instantly searchable collection of upstream technology portals, streaming testbeds, and experimental domain endpoints.

Unlike traditional bookmark managers or link dump repositories, this project treats every external URL as a first‑class data record with metadata, availability probes, and categorical tags. The index is rebuilt on every pull request, ensuring that stale or redirected entries are automatically flagged. The primary user base includes edge network engineers, content accessibility researchers, and open‑source intelligence (OSINT) practitioners who require reproducible link sets for benchmarking, filtering, or regional availability testing.

The repository does not host, proxy, or mirror any third‑party content. It provides a structured manifest and a set of verification scripts that allow users to validate the responsiveness of each endpoint from their own network vantage point. The project is intentionally minimal on runtime dependencies, focusing on POSIX‑compatible shell scripts and Python 3.11+ standard library modules.

## 功能概览

- **Structured Link Manifest** – The master index is maintained as a YAML file with fields for URL, category, last‑seen status, and optional notes, enabling automated parsing and integration into monitoring pipelines.

- **Availability Probe Suite** – A lightweight Python script performs concurrent HEAD requests with configurable timeouts and retry logic, generating a machine‑readable JSON report of HTTP status codes, response times, and TLS certificate expiry warnings.

- **Categorical Tagging System** – Each URL is assigned one or more predefined tags (e.g., streaming, video, research, fallback) to facilitate filtered exports and scenario‑specific sub‑lists.

- **Automated PR Validation** – Every pull request that adds, removes, or modifies an entry triggers a CI workflow that validates YAML syntax, checks for duplicate URLs, and runs a connectivity test against all changed endpoints.

- **Static HTML Table Generator** – An optional build step produces a plain HTML page that renders the index as a sortable, searchable table, suitable for internal dashboards or offline documentation servers.

- **Export Adapters** – The index can be exported as plain text line‑by‑line, CSV for spreadsheet analysis, or as a Prometheus‑style metric exposition format for integration with monitoring stacks.

- **Atomic Update Logging** – Every modification to the index is automatically logged with a timestamp and the committer’s handle, providing a complete audit trail for compliance or forensic purposes.

## 应用场景

- **Edge Network Benchmarking** – Network engineers can use the index to periodically measure reachability and latency to a diverse set of streaming and video endpoints from different geographic PoPs, helping to validate BGP routing policies and CDN selection logic.

- **Content Filtering Rule Testing** – Security researchers and firewall administrators may employ the URL list as a test suite to verify that corporate or national filtering policies are working as intended, without relying on proprietary or frequently changing blacklists.

- **Regional Availability Monitoring** – Operations teams can deploy the probe script across multiple cloud regions or on‑premise agents to detect regional outages or DNS poisoning events, with the results fed into a centralized time‑series database like Prometheus or InfluxDB.

- **Open‑Source Intelligence (OSINT) Workflows** – Analysts can incorporate the index into their daily reconnaissance routines, using the categorised links as starting points for investigating new domain patterns, TLD usage, or infrastructure shifts in specific content verticals.

- **CI/CD Pipeline Quality Gates** – Development teams can add the probe script as a lightweight health check step in their deployment pipelines, ensuring that all external service dependencies referenced in their documentation remain responsive before a release is finalized.

## 快速开始

Clone the repository, install the minimal dependencies, and run the verification script to test all listed endpoints. The following steps assume a Debian‑based Linux environment with Python 3.11 or later.

```bash
git clone https://github.com/unchained-index/project-unchained.git
cd project-unchained
pip install -r requirements.txt --user
python3 probe.py --manifest manifest.yaml --report report.json --timeout 5 --concurrency 20
```

For a quick sanity check that only tests a single URL, use:

```bash
python3 probe.py --single https://example.com --verbose
```

To generate the static HTML table from the current manifest, run:

```bash
python3 build_html.py --manifest manifest.yaml --output index.html
```

## 贡献指南

We welcome contributions that improve the reliability, coverage, or usability of the index. All changes must follow the established YAML schema and pass the automated validation pipeline.

1. Fork the repository and create a new feature branch from `main`. Name your branch according to the change type, e.g., `add-streaming-endpoint` or `fix-typo-manifest`.

2. Edit the `manifest.yaml` file to add, remove, or update URL entries. Each entry must include the `url` field, at least one `category`, and an optional `note` describing the endpoint’s purpose or origin.

3. Run the local validation suite to catch syntax errors and duplicate entries before committing. Use `pytest tests/` to execute the full test suite, or run `python3 probe.py --manifest manifest.yaml --dry-run` to check manifest parsing without sending network requests.

4. Commit your changes with a clear, descriptive message that references the relevant issue number if applicable. Push your branch and open a pull request against the `main` branch of the upstream repository.

5. Wait for the CI checks to complete. At least one maintainer approval is required for merging. If the CI flags any connectivity failures for new endpoints, please verify the endpoint manually and update the `note` field with the observed status.

## 常见问题

**Q: Why does the probe script report a timeout for some URLs while they work in my browser?**

A: The probe uses a conservative default timeout of 5 seconds with no retries. Some endpoints may perform client‑side content negotiation, require specific user‑agent headers, or employ anti‑bot mechanisms that delay the initial response. You can increase the timeout using the `--timeout` flag. Additionally, the script does not execute JavaScript or render pages – it only checks the availability of the TCP connection and the HTTP status line. If an endpoint relies on complex redirection chains or lazy loading, consider using the `--follow-redirects` flag (enabled by default) and ensure your network allows outbound HTTPS traffic on port 443.

**Q: How often is the manifest updated, and how can I report a broken link?**

A: The manifest is updated on a best‑effort basis, typically in response to user reports or automated scans. We do not guarantee real‑time freshness. To report a broken link, please open a GitHub issue with the URL, the observed HTTP status code, and the time of your test. Alternatively, you can submit a pull request that removes or comments out the malfunctioning entry. We encourage users to run the probe script themselves and attach the JSON report to the issue for faster diagnosis.

**Q: Can I use this index for commercial monitoring or SLA verification?**

A: Yes, the project is released under the MIT license, which allows both private and commercial use without restriction. However, we explicitly disclaim any warranty regarding the accuracy or uptime of the listed third‑party endpoints. The manifest is provided “as is” and should be treated as a reference dataset for testing and development, not as a production‑grade monitoring solution. For high‑stakes environments, we recommend supplementing this index with dedicated commercial monitoring services and more frequent polling intervals tailored to your specific requirements.
