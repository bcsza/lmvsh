# VaultLink

VaultLink 是一个面向数字内容研究者与媒体归档从业者的技术资源导航与外部链接聚合系统。该项目不提供任何实质内容存储服务，而是作为结构化入口层，对互联网中分散的媒体资源站点进行编目、分类与功能说明展示。项目定位为技术性外链汇总站，帮助用户快速定位特定域名下的服务类型、可用协议与内容倾向，降低信息检索成本。目标用户包括网络资源整理者、自动化采集脚本开发者、学术研究人员以及关注数字媒体存档的技术人员。

## 功能概览

- **域名编目管理**：支持手动录入与批量导入外部域名，自动提取协议头、子域名与顶级域信息，生成标准化的条目记录。
- **服务类型标签化**：每个链接可标记为主站、镜像站、图片托管、视频索引、备用入口等分类标签，便于过滤筛选。
- **可用性检查**：提供轻量级 HTTP/HTTPS 探活机制，定期检测每个域名的可达状态与响应码，辅助判断服务可用性。
- **元数据备注系统**：支持为每个链接添加技术备注，如 CDN 使用情况、是否限制 Referer、是否支持范围请求等字段。
- **批量导出与分享**：支持将当前编目列表导出为 JSON、CSV 或纯文本格式，便于集成至下游数据处理流水线。
- **静态页面生成**：内置模板引擎，可生成只读的静态导航页，适合部署在内网或文档站点中供团队查阅。
- **变更日志追踪**：记录每次条目增删改的操作时间与摘要，便于回溯配置变更历史。

## 应用场景

- **数字媒体资源整理**：内容策展人可使用 VaultLink 对分散在多个域名的影视预告片、独立短片或公开课视频来源进行统一编目，形成可维护的资源清单，避免收藏夹散乱。
- **自动化采集任务配置**：数据采集工程师可将本系统作为种子 URL 源，通过 API 或导出功能获取最新域名列表，动态更新爬虫起始节点，减少人工维护配置文件的工作量。
- **学术引用与来源备案**：研究人员在分析特定类型的在线媒体分布情况时，可利用本系统记录观察样本集，确保研究可复现性，并为后续追踪域名变更提供基线参照。
- **内网镜像站点导航**：企业或教育机构内网若存在多个媒体缓存或代理入口，可通过 VaultLink 搭建内部导航页，统一公布可用入口地址，降低终端用户查找成本。

## 快速开始

以下指令适用于 Linux/macOS 环境，需提前安装 Git 与 Node.js 16.x 以上版本。

```bash
# 克隆仓库
git clone https://github.com/vaultlink/vaultlink.git
cd vaultlink

# 安装依赖
npm install

# 启动开发服务器（默认端口 3000）
npm run dev
```

生产环境构建与启动：

```bash
npm run build
npm start
```

## 贡献指南

1. 查阅问题列表：访问 GitHub Issues 页面，了解当前待解决的缺陷或功能请求，选择未被认领的任务。对于新需求，建议先创建讨论议题以明确设计方向。
2. 派生仓库并创建特性分支：从主仓库派生个人副本，然后基于 main 分支创建以 feature/ 或 fix/ 为前缀的新分支，避免直接在主分支上修改。
3. 编写代码与测试：遵循项目现有代码风格（ESLint 配置），为核心逻辑补充对应的单元测试，确保所有测试用例通过后方可提交。
4. 提交变更并推送：提交信息采用约定式提交格式（如 feat: 新增批量导入去重逻辑），推送至个人派生仓库后发起 Pull Request，描述中需清晰说明变更内容与影响范围。
5. 评审与合并：项目维护者将在 3 个工作日内进行评审，若需调整会通过评论反馈。通过后由维护者完成合并。

## 常见问题

**问：VaultLink 是否存储或缓存外部站点的实际媒体内容？**

答：否。本系统仅存储用户输入的域名文本、分类标签与可选备注，不发起除可用性探活外的任何数据请求，不下载、不代理、不缓存任何媒体文件。探活仅检查 HTTP 状态码，不解析响应体。

**问：如何更新已收录域名的标签或备注信息？**

答：可通过 Web 管理界面（若启用）或 CLI 命令行工具执行更新操作。CLI 示例：`vaultlink update --domain meitianfilm.lol --tag "video-archive"`。所有变更记录会写入变更日志表。

**问：静态页面生成模式是否需要数据库？**

答：生成静态页面前需要数据库中存在有效数据。生成过程读取数据库并输出纯 HTML/CSS/JS 文件，生成的页面无需数据库支持即可独立部署，适合只读展示场景。
