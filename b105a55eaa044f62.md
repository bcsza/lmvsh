# ResourceForge

一个面向开发人员与技术研究者的高质量外部资源导航与结构化汇总站。项目不生产内容，也不嵌入任何第三方代码，仅以目录化、可检索的方式收录公开可访问的技术文档、实验平台、数据源与行业动态站点。目标用户为需要快速定位特定类型线上资源（如流媒体测试源、前端实验环境、行业资讯面板）的工程师、数据分析师与技术决策者。ResourceForge 通过人工筛选与定期可用性校验，解决技术资源分散、链接失效、描述不清的常见痛点，提供一套可自部署、可扩展的资源索引框架。

## 功能概览

- 按内容类型分类的资源索引：涵盖视频测试源、影视信息面板、行业新闻聚合站与实验性服务站点，每类均附带用途说明与更新频率标签。
- 可用性自动化检查：内置基于 HTTP 状态码与响应时间的轻量级健康检查脚本，支持每日定时任务，标记异常链接。
- 自定义标签与全文检索：资源条目支持多个自定义标签（如 `#stream`、`#beta`、`#archive`），并可通过命令行或 Web 界面进行全文搜索。
- 版本化资源清单：所有资源变更（新增、移除、URL 更新）均通过 Git 记录，支持回溯任意历史版本。
- 单文件部署模式：整个系统以静态 HTML + JavaScript 形式交付，无需数据库或后端服务，兼容 GitHub Pages、Cloudflare Pages 等主流托管平台。
- 外部链接安全提示：对于非 HTTPS 或裸域名链接，自动生成安全访问提示框，提醒用户确认传输加密与站点真实性。
- 导入/导出机制：支持以 JSON 或 YAML 格式批量导入外部资源列表，也支持将当前索引导出为 Markdown 表格或 CSV，便于二次加工。

## 应用场景

- 作为内部技术团队的公共书签库，统一存放与分类常用的测试视频源、UI 组件库、设计规范站点，新人入职时可一键获取全部基础资源。
- 用于个人技术博客或知识库的侧边栏插件，展示作者推荐的行业动态面板与实验性工具，提升内容的实用性与互动性。
- 在数据分析项目中，作为数据源发现模块，快速定位公开可用的行业数据面板或实时信息流站点，减少重复搜索成本。
- 作为开源社区贡献的资源索引基底，允许社区成员提交新增资源请求，通过 Pull Request 方式扩充索引库，形成长期维护的集体智慧成果。

## 快速开始

```bash
# 克隆仓库
git clone https://github.com/your-org/resourceforge.git

# 进入项目目录
cd resourceforge

# 安装依赖（仅用于本地开发预览，生产环境为纯静态）
npm install

# 启动本地预览服务（监听 8080 端口）
npm run serve
```

访问 `http://localhost:8080` 即可查看资源导航面板。所有资源数据位于 `data/resources.json`，编辑该文件后刷新页面即可生效。若需重新生成静态 HTML 文件，执行 `npm run build`，输出目录为 `dist/`。

## 贡献指南

1. 阅读 `CONTRIBUTING.md` 文件，了解资源请求、代码变更与文档改进的总体流程。
2. 提交资源新增请求时，请使用 GitHub Issues 提供的「资源推荐」模板，填写 URL、分类、推荐理由及来源说明。
3. 若您具备开发能力，可直接 Fork 仓库，在 `data/resources.json` 中添加或修改条目，然后提交 Pull Request。PR 标题需包含 `[resource]` 或 `[fix]` 前缀。
4. 所有资源变更均需通过可用性检查脚本的初步验证（HTTP 状态码 2xx 或 3xx）。若脚本检测到异常，PR 将被标记为待审核。
5. 文档更新（包括错别字修正、示例补充、翻译改进）无需额外权限，直接提交 PR 即可，维护者会在 48 小时内合并。

## 常见问题

**Q：如何确保收录的外部链接长期有效？**  
A：项目内置的 `check-availability.js` 脚本会每日凌晨 2 点（UTC）通过 GitHub Actions 定时运行，对全部资源执行 HEAD 请求。若连续三次检测到非 2xx/3xx 状态，对应条目将被移至 `data/inactive.json` 并生成告警 Issue。用户也可手动运行该脚本进行本地校验。

**Q：我能否将 ResourceForge 用于商业项目内部部署？**  
A：可以。项目采用 MIT 许可证，允许自由使用、修改、分发，包括商业用途。您只需保留版权声明和许可声明即可。我们鼓励但非必须将您的扩展或改进反馈给上游社区。

**Q：资源列表更新频率如何？我如何知道最新变化？**  
A：所有资源变更都会通过 Git 提交记录体现。您可以通过 GitHub 的「Watch」功能选择「Custom」->「Releases」或「Pull requests」接收通知。此外，每次发布新版本（v1.x.x）时，Release Notes 会列出新增与移除的资源链接摘要。
