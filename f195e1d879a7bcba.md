# WebLink Navigator

WebLink Navigator 是一个面向数字内容研究者、网络资源整理者以及技术开发人员的综合性网络资源导航与状态监测工具。该项目定位于解决网络资源分散、链接失效难以追踪、多源数据整合困难等实际问题，通过结构化的数据采集、状态监控与可视化呈现，帮助用户高效管理和利用网络上的公开资源。

项目采用模块化设计，核心引擎支持定时抓取、状态码验证、响应时间记录以及变更检测，并对外提供清晰的 RESTful API 与轻量级管理面板。WebLink Navigator 不依赖特定商业服务，完全基于开源技术栈构建，可私有化部署于 Linux、macOS 或 Windows 环境。

## 功能概览

- **多协议资源采集**：支持 HTTP/HTTPS 协议下的页面内容抓取与资源元数据抽取，自动识别字符集与内容类型。
- **链路状态实时监控**：对配置的资源列表进行周期性可用性探测，记录状态码、响应耗时、SSL 证书有效期等关键指标。
- **结构化存储与版本管理**：将采集结果以 JSON 及 SQLite 格式持久化，自动生成资源变更的历史版本差异报告。
- **灵活的任务编排系统**：基于 Cron 表达式配置采集与监控任务，支持并发执行与失败重试策略。
- **轻量级管理 API**：提供基于 Token 鉴权的管理接口，支持资源增删改查、任务启停及状态快照查询。
- **数据导出与集成能力**：支持将监控数据导出为 CSV、Markdown 或 HTML 摘要报表，便于对接第三方分析平台。
- **内置模板引擎**：提供默认的响应式状态展示页面，同时支持用户自定义模板以呈现资源列表。

## 应用场景

1.  **个人书签的自动化健康检查**：用户可将自己的收藏夹或常用资源列表导入系统，每日定时检查各链接是否仍然有效，及时发现死链或异常跳转，避免收藏内容失效。
2.  **内容聚合站点的源管理**：对于运营内容聚合或导航站点的开发者，可使用 WebLink Navigator 统一管理外部引用源，监控源站点的可用性与内容更新频率，确保聚合内容的可靠性。
3.  **网络审计与合规性追踪**：企业或教育机构可利用该工具对内部文档中引用的外部资源进行定期合规性扫描，记录资源变更历史，满足审计对引用来源可追溯性的要求。
4.  **技术调研的辅助工具**：在研究特定领域（如在线视频、文档分享、学术资源）的网络服务分布时，可通过项目内置的批量采集与状态分析功能，快速获取资源存活率、地域响应差异等数据，辅助决策。

## 快速开始

以下命令适用于 Linux / macOS 环境，Windows 用户可使用 Git Bash 或 WSL 执行。

```bash
# 克隆项目仓库
git clone https://github.com/weblink-navigator/weblink-navigator.git
cd weblink-navigator

# 安装依赖（使用 Python 虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化数据库并运行内置示例资源监控
python manage.py initdb
python manage.py add-resource --file samples/initial_resources.json
python manage.py run-monitor --once
python manage.py runserver --host 0.0.0.0 --port 8080
```

启动成功后，访问 `http://localhost:8080/dashboard` 可查看默认管理面板。

## 常见问题

**Q：监控任务执行时出现 SSL: CERTIFICATE_VERIFY_FAILED 错误，如何解决？**

A：此问题通常由目标站点 SSL 证书过期或使用自签名证书引起。可在配置文件中设置 `VERIFY_SSL = False` 以跳过验证（仅建议在内网或测试环境使用）。对于生产环境，推荐将根证书或目标站点证书添加到系统信任链，并在 fetcher 中指定 `cert_path` 参数。

**Q：项目是否支持对需要登录或带有反爬机制的页面进行监控？**

A：基础版本仅支持无状态 HTTP 请求。对于需要携带 Cookie 或特定 Header 的资源，可在资源配置中添加 `headers` 和 `cookies` 字段，fetcher 模块会自动合并。对于 JavaScript 渲染的页面，建议结合 Playwright 或 Selenium 扩展采集器，项目已预留 `custom_fetcher` 接口供用户注入自定义抓取逻辑。

**Q：如何迁移已有的资源列表（如书签 HTML 或 CSV 文件）？**

A：项目提供了 `scripts/import_bookmarks.py` 辅助脚本，支持将 Netscape 格式的书签导出文件或简单的两列 CSV（标题, URL）转换为项目标准的 JSON 资源配置。具体使用方法请参阅 `docs/import_guide.md`。
