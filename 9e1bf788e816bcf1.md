# NEXUS-MCP Resource Gateway

NEXUS-MCP 是一个面向开发者、技术研究员与内容创作者的轻量级资源网关与导航工具，专注于聚合、分类与快速检索互联网中的优质技术文档、社区讨论、工具链与媒体资源。项目定位为“技术外链的结构化索引系统”，通过静态数据与轻量后端结合，帮助用户在复杂信息环境中快速定位高价值内容，避免重复检索与信息过载。

目标用户包括开源贡献者、运维工程师、AI 应用开发者以及需要频繁查阅外部技术资料的知识工作者。项目本身不存储任何第三方内容，仅提供 URL 元数据管理与分类展示能力，所有外链以只读方式呈现，确保合规与安全。

---

## 功能概览

- **多级分类导航**：支持按技术领域、资源类型、热度标签进行三级分类筛选，便于快速缩小检索范围。
- **链接状态监控**：定期对收录 URL 进行可用性探测，自动标记异常链接，降低无效访问率。
- **全文元搜索**：基于标题、描述、关键词对已收录资源进行轻量级全文检索，返回相关度排序结果。
- **自定义收藏夹**：用户可导入/导出个人链接收藏列表，支持 JSON 与 CSV 格式。
- **访问统计看板**：提供各资源被点击次数的聚合统计，辅助判断资源热度。
- **暗色主题与阅读模式**：适配不同使用环境，减少长时间浏览时的视觉疲劳。
- **RSS 订阅源生成**：为每个分类目录生成标准 RSS 2.0 订阅链接，便于集成进其他阅读器。
- **一键导入/导出配置**：支持通过配置文件批量添加新资源组，便于团队内部共享导航结构。

## 应用场景

1. **技术团队内部知识库导航**  
   开发团队可将 NEXUS-MCP 部署为内部文档入口，统一聚合 API 文档、设计规范、CI/CD 配置指南等常用链接，减少新成员上手时的信息查找成本。

2. **开源项目 README 外链补充**  
   开源维护者可将项目相关的依赖文档、社区论坛、示例代码仓库等资源通过 NEXUS-MCP 集中管理，并在 README 中引用该网关地址，避免主文档过长。

3. **技术写作与内容策展**  
   博客作者或技术编辑可使用本工具维护个人“灵感池”，分类保存文章参考资料、数据来源与配图资源，并在写作时快速调取。

4. **教育培训中的课程资料索引**  
   讲师可将课程涉及的在线实验环境、视频教程、习题答案链接统一收录，学生通过单一入口访问全部外部资源，降低跳转混乱风险。

## 快速开始

以下命令适用于 Linux / macOS / WSL 环境，Windows 原生用户建议使用 Git Bash 或 PowerShell 7。

```bash
# 1. 克隆仓库
git clone https://github.com/nexus-mcp/gateway.git
cd gateway

# 2. 安装依赖（使用 pip 虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 3. 初始化本地数据库
python scripts/init_db.py --seed data/default_seed.json

# 4. 启动开发服务
python app.py --host 127.0.0.1 --port 8080
```

访问 `http://127.0.0.1:8080` 即可进入本地实例。生产部署请参考 `docs/deployment.md` 使用 Gunicorn + Nginx 组合。

## 常见问题

**Q：项目是否存储用户上传的文件或第三方内容？**  
A：不存储。NEXUS-MCP 仅保存链接的元数据（标题、描述、分类、URL 字符串），所有外部内容均实时从原始站点获取。用户添加资源时仅需提供链接与描述，系统不会缓存或代理任何媒体文件。

**Q：如果某个收录链接失效，系统如何处理？**  
A：后台定时任务（默认每 6 小时执行一次）会发送 HEAD 请求检测链接可用性，连续三次失败则标记为 `unstable` 状态，并在前端用视觉标识提示用户。管理员可手动触发重新检测或直接删除失效条目。

**Q：能否部署在无外网环境的内网中？**  
A：可以。项目本身不依赖任何外部 CDN 或在线服务（除用户主动添加的链接外）。所有前端库（如 Chart.js 轻量版）均已打包在 `static/assets/` 中，SQLite 数据库无需网络，Redis 可配置为关闭状态，完全支持离线内网部署。
