# NexusIndex

一个面向中文互联网内容创作者、数据研究者与信息获取者的轻量级技术资源聚合与导航系统。项目通过人工筛选与自动化可用性检测相结合的方式，维护一份高质量、高可用性的流媒体与信息源外链索引表，解决个人用户在海量低质站点中反复试错、链接频繁失效以及跨区域访问受阻的核心痛点。

NexusIndex 不提供任何实体内容托管服务，亦不存储、缓存或转发任何受版权保护的音视频文件。项目本质是一个结构化的 URL 状态监控与分类目录，其目标用户包括需要稳定信息采集入口的数据工程团队、关注跨境内容分发的产品经理以及希望减少无效浏览时间的普通技术爱好者。

## 功能概览

- **多源异构链接聚合**：统一收纳涵盖影视、学术、资讯等不同维度的网络资源入口，支持按域名、标签与最后验证时间进行多级筛选与排序。

- **自动可用性心跳检测**：内置基于 Python asyncio 的异步 HTTP 探针，每 6 小时对所有收录域名执行 TLS 握手、响应码与响应体关键字三重校验，实时标记异常节点。

- **访问策略智能推荐**：依据用户 IP 归属区域与目标域名的历史响应延迟，动态提示当前最优访问方式（直连、代理或镜像站），降低非必要网络开销。

- **结构化元数据输出**：支持将索引数据导出为 JSON、CSV 及 Prometheus 格式的 Metrics 端点，便于接入 Grafana 监控面板或下游数据清洗管道。

- **去重与失效自动剔除**：每日定时任务比对新增链接与存量库的域名指纹，自动合并重复条目，并将连续 3 次探测失败的链接迁移至归档表，保持主表数据鲜活。

- **轻量级 Web 管理界面**：提供基于 Flask 的后台仪表板，支持手动触发探测、编辑备注标签以及查看各域名的历史可用性趋势图。

- **开放 API 查询接口**：暴露 RESTful 风格的 `/v1/domains`、`/v1/status` 等端点，允许第三方应用按类别、可用率或响应时间检索链接列表，方便二次开发集成。

## 应用场景

- **数据采集管道前置治理**：数据工程团队可将 NexusIndex 部署为内部调度系统的上游配置源，每日自动拉取最新存活域名列表，动态更新 Scrapy 或 Nutch 爬虫的起始种子池，显著降低因链接失效导致的任务异常中断率。

- **跨国业务内容合规验证**：出海企业运维人员可利用本项目的区域延迟探测功能，定期评估不同国家与地区对特定域名的可访问性，为 CDN 选型、边缘节点配置以及容灾切换策略提供客观数据支撑。

- **个人知识库信息源维护**：独立博客作者与研究员可将项目用作个人书签管理的替代方案，通过标签体系对收藏链接进行精细化分类，并借助过期提醒功能及时清理或更新失效的参考文献来源。

- **网络诊断辅助工具**：网络爱好者可利用项目提供的单次探测命令行工具，快速验证特定域名在本地的真实解析与响应状况，辅助排查运营商劫持或 DNS 污染等异常现象。

## 快速开始

以下指令适用于 Linux / macOS / Windows WSL 环境，要求已安装 Python 3.9 及以上版本与 Git。

```bash
# 克隆项目仓库至本地
git clone https://github.com/nexusindex/nexusindex.git
cd nexusindex

# 创建 Python 虚拟环境并激活
python3 -m venv venv
source venv/bin/activate     # Windows 请使用 venv\Scripts\activate

# 安装所有生产依赖
pip install --upgrade pip
pip install -r requirements.txt

# 初始化 SQLite 数据库表结构并执行首次全量探测
python manage.py initdb
python manage.py probe --full

# 启动开发调试服务器
flask run --host=0.0.0.0 --port=5000
```

启动成功后，访问 `http://localhost:5000` 即可查看当前收录的所有链接状态面板。生产环境部署建议配合 Gunicorn 与 Nginx 使用。

## 常见问题

**问：项目自身会存储或播放任何视频文件吗？**

答：不会。NexusIndex 仅维护域名与 URL 的文本列表及其可用性元数据（如响应时间、HTTP 状态码）。项目不涉及任何文件上传、下载、转码或流媒体分发行为，所有链接均直接指向第三方站点，用户点击后将离开本系统。

**问：探测结果中的“异常”状态代表目标站点一定无法访问吗？**

答：不一定。探测结果受网络环境、目标站点的反爬策略、临时维护以及地域性封锁等多重因素影响。单个探测节点的失败结果可能不代表全网不可达。建议用户结合自身实际访问体验，并通过项目提供的“手动验证”功能进行二次确认。

**问：我可以将本项目部署在公网作为公共服务对外提供吗？**

答：可以。但请注意，本项目的核心价值在于链接管理工具本身，而非链接指向的具体内容。作为部署方，您应遵守所在国家与地区的法律法规，并建议在页面上明确声明所有外部链接均非本服务提供，最终访问风险由使用者自行承担。
