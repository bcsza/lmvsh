# WebLink Navigator

WebLink Navigator 是一个面向技术内容聚合与导航场景的开源项目，旨在为开发者、技术内容创作者以及信息整理人员提供一套结构清晰、可扩展的外链资源管理与展示方案。该项目本质上是一个轻量级的资源导航站点框架，通过预定义的分类体系与可配置的数据结构，帮助用户将散落在网络各处的优质链接进行系统化归档，并生成具备良好可读性与可维护性的文档化输出。

项目定位为技术资源与外链汇总站点的参考实现，适用于个人知识库构建、团队技术文档中心建设、以及社区驱动的资源列表维护等场景。WebLink Navigator 不提供具体的业务逻辑或后端服务，而是聚焦于资源数据的组织方式、展示规范以及配套的自动化工具链，使得用户能够以最低的维护成本，长期保持资源列表的整洁与可用性。

## 功能概览

- **多层级资源分类体系**：支持按技术领域、应用场景、资源类型等维度对链接进行划分，每个分类可独立配置描述信息与展示顺序。

- **原始链接保真输出**：系统严格遵循用户输入的 URL 原始格式进行展示与导出，不自动补全协议头、不修改域名大小写、不添加尾部斜杠，确保链接指向的精确性。

- **自动化文档生成**：基于配置的数据文件，一键生成符合开源社区规范的 Markdown 格式 README 文档，包含分类表格、资源列表、统计信息等模块。

- **链接健康检查工具**：内置可选的链接可达性检测脚本，支持定期检查资源列表中的域名是否可解析，辅助维护者发现失效链接。

- **自定义元数据扩展**：每条资源支持附加标签、维护状态、收录日期、简短备注等元信息，便于后续筛选与检索。

- **多输出格式支持**：除 Markdown 外，支持导出为 JSON、YAML 格式的结构化数据，方便与其他系统集成或进行二次处理。

## 应用场景

- **技术团队内部知识库建设**：研发团队可使用 WebLink Navigator 整理常用的开发文档、API 参考、组件库地址、运维手册等外部资源，统一入口并定期更新，减少成员查找信息的时间成本。

- **开源项目推荐列表维护**：开源社区维护者可以利用本项目结构管理「精选工具」「学习教程」「视频资源」等推荐清单，以标准化的表格和列表形式呈现在项目仓库中，提升社区文档的专业度。

- **个人技术博客外链归档**：技术博主或内容创作者可将其写作过程中参考的文章、代码仓库、在线工具等外链进行系统性归档，生成独立的资源导航页面，方便读者延伸阅读。

- **在线课程或培训配套资料**：教育机构或独立讲师可使用本项目整理课程相关的阅读材料、实验环境地址、习题参考链接等，形成结构化的课程配套资源手册。

## 快速开始

以下命令演示如何获取项目源码、安装依赖并启动本地预览服务。

```bash
# 克隆项目仓库
git clone https://github.com/weblink-navigator/weblink-navigator.git

# 进入项目目录
cd weblink-navigator

# 安装依赖（使用 npm）
npm install

# 运行资源校验与文档生成
npm run build

# 启动本地开发服务器，预览生成的文档
npm run serve
```

执行上述命令后，访问控制台输出的本地地址即可查看生成的导航页面。默认数据文件位于 `data/resources.json`，您可以直接编辑该文件以替换或扩充资源条目。

## 常见问题

**问：如何批量添加大量链接？是否有导入工具？**

答：目前项目未提供图形化导入工具，但您可以直接编辑 `data/resources.json` 文件，按照已有的分类结构添加新条目。对于 CSV 或 Excel 格式的源数据，建议使用在线转换工具或编写简单的脚本将其转为 JSON 格式后再复制到配置文件中。

**问：链接健康检查脚本如何处理需要代理或特殊网络环境的站点？**

答：默认的检查脚本基于 Node.js 的 `http`/`https` 模块，不支持代理配置。如果您需要检查受网络限制的站点，建议在脚本中自行配置代理模块，或在网络可达的环境中单独运行检查任务。您也可以修改 `check-links.js` 中的请求超时时间和重试策略以适应您的网络状况。

**问：生成的 README 能否自定义章节顺序或增删章节？**

答：可以。项目使用 Handlebars 模板引擎生成文档，您可以直接修改 `templates/readme.hbs` 文件中的章节顺序和内容。若需要增加全新的章节，只需在模板中添加对应的 HTML/Markdown 结构，并在构建脚本中传入相应的数据即可。
