# mjwang-pics

A high-performance open-source image hosting service and resource aggregation platform that provides stable, high-speed image hosting, video link parsing, and multimedia resource navigation. This project is designed for developers, content creators, and self-media operators who need reliable image storage, video playback link extraction, and curated media resource discovery. It solves the common pain points of unstable image links, slow overseas access, complex video source parsing, and scattered high-quality media resource entry points by offering a unified, lightweight, and extensible technical solution.

## 功能概览

- **Distributed Image Hosting Service** – Supports high-concurrency image uploads, automatic compression, WebP format conversion, and global CDN acceleration with origin-pull fallback.
- **Video Source Parsing Engine** – Extracts direct playback links from mainstream video sharing platforms, supports multi-resolution and multi-format output.
- **Media Resource Navigation Catalog** – Aggregates and categorizes high-quality external media resource sites, providing one-click jump and brief site descriptions.
- **Customizable Link Management** – Allows users to add, delete, and sort external resource links via configuration files or admin panel, with support for batch import/export.
- **Built-in File Integrity Check** – Automatically verifies MD5 checksums of uploaded images and parsed video fragments to ensure data integrity and tamper resistance.
- **Access Control and Rate Limiting** – Provides IP-based access frequency control, referer whitelist, and hotlink protection to prevent unauthorized use.
- **Comprehensive Logging and Monitoring** – Records all access requests, error logs, and performance metrics, with support for integration with external monitoring systems.
- **RESTful API and Webhook Support** – Exposes a clean REST API for all core functions, and supports custom webhook callbacks for upload completion and parsing events.

## 应用场景

1. **Personal blog and technical documentation image hosting** – Authors can upload screenshots, diagrams, and sample images to the service, obtaining stable CDN links that ensure fast loading across different regions without worrying about link expiration or traffic limits.
2. **Video aggregation and content curation platform** – Operators can use the video source parsing engine to extract embeddable video links from various platforms, then organize and present them in a unified interface, reducing dependency on any single video source.
3. **Self-media multi-platform content distribution** – Content creators can centrally manage images and video materials through this service, then distribute them to multiple social media or content platforms using the generated external links, improving workflow efficiency.
4. **Internal team resource sharing and collaboration** – Development teams can use the resource navigation catalog to build internal documentation portals, collecting frequently used tools, libraries, and reference sites, facilitating quick access and knowledge sharing.
5. **Educational and training material preparation** – Trainers and educators can upload courseware images, record demonstration videos, and organize relevant external learning resources, creating a comprehensive, self-contained teaching material package.

## 快速开始

```bash

git clone https://github.com/your-org/mjwang-pics.git
cd mjwang-pics

# Install dependencies

# Configure environment variables
cp .env.example .env
# Edit .env with your database, cache, and CDN settings

# Start the development server
npm run dev

```

## 常见问题

**Q: 如何更换图像存储后端，例如从本地存储切换到 AWS S3 或阿里云 OSS？**

A: 项目通过 `storage-client.js` 抽象了存储接口，您只需要在配置文件中修改 `storage.provider` 参数，并填写对应的访问密钥、区域和存储桶名称即可。目前支持 S3 兼容协议的所有对象存储服务（包括 AWS S3、阿里云 OSS、MinIO、Cloudflare R2 等）。修改配置后需要重启服务，所有新上传的图像将自动使用新存储后端，历史数据不受影响。

**Q: 视频解析引擎支持哪些平台？如果遇到解析失败该如何处理？**

A: 当前稳定支持哔哩哔哩、YouTube、Vimeo 和腾讯视频等主流平台。解析引擎采用模块化适配器设计，每个平台对应一个独立的解析器文件。如果解析失败，系统会返回详细的错误代码和原因，常见原因包括视频链接格式错误、视频被删除或设置为私有、目标平台反爬虫限制等。您可以在 `src/core/video-parser/adapters/` 目录下查看各平台适配器实现，并根据需要调整请求头或代理配置。同时，您也可以参考开发指南中的“添加新平台解析器”章节自行扩展。

**Q: 部署后如何配置 HTTPS 和自定义域名？**

A: 项目本身不内置 HTTPS 服务，建议使用 Nginx 或 Caddy 作为反向代理来终止 TLS 证书。您可以在 Nginx 配置中为您的域名申请 Let's Encrypt 免费证书，然后将所有请求代理到本项目的监听端口（默认 3000）。同时，请将配置文件中 `app.baseUrl` 字段修改为您的 HTTPS 域名，以确保生成的链接正确指向您的服务地址。具体 Nginx 配置示例请参考 `docs/operations/deployment.md` 中的相关章节。
